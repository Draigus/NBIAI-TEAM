# Studio Brain Sprint -- Engagement Template

## What It Is

A fixed-price, 2-3 week engagement where NBI audits a game studio's knowledge landscape, compiles their institutional knowledge into a structured Studio Brain, creates role-specific knowledge bases, and trains an internal champion to maintain the system.

## The Problem It Solves

Every game studio has critical institutional knowledge scattered across Confluence, Google Docs, Slack, email, and people's heads. When key people leave, that knowledge walks out the door. When new hires start, they spend months rebuilding context. When AI tools are adopted, they hallucinate because they lack studio-specific context.

## Deliverables

1. **Studio Brain** -- A compiled, structured markdown knowledge base covering the studio's identity, products, team, processes, and decisions. Built using NBI's /compile-client methodology with provenance tags on every fact.
2. **Role Knowledge Bases (3-5)** -- Domain-specific knowledge assets for the studio's key functions (typically: Production, Game Design, Tech, Art, QA). Each is a 150-250 line structured reference document.
3. **Claude Code Configuration** -- A configured CLAUDE.md file with routing tables, approval gates, and quality standards tailored to the studio's workflow.
4. **Champion Training** -- 2-hour session with an internal champion covering: how to maintain the Brain, how to add new role knowledge, how to use routing tables, and how to run the system audit.
5. **30-Day Check-in** -- One follow-up session at 30 days to review Brain health, answer questions, and tune the configuration.

## Delivery Methodology

- **Days 1-3:** Document collection and audit. Glen reviews the studio's knowledge landscape: Confluence, Google Docs, Slack, email archives, meeting notes. Identifies what exists, what's scattered, and what's missing.
- **Days 4-7:** Brain compilation. Run /compile-client on collected documents. Verify facts, resolve contradictions, establish provenance. Build the Studio Brain with sections mirroring NBI's own Brain structure.
- **Days 8-10:** Role knowledge base creation. For each key function, create an AGENT.md-style knowledge asset using NBI's role templates. Populated with studio-specific benchmarks, processes, and domain knowledge.
- **Days 11-12:** Configuration and testing. Set up CLAUDE.md with routing tables mapping topics to knowledge bases. Configure approval gates appropriate to the studio's decision-making structure. Test with real queries.
- **Days 13-15:** Training and handoff. Train the internal champion. Deliver all artefacts. Run the initial system audit to establish baseline scores.
