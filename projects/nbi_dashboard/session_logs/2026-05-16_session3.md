### Sidebar Toggle — Root Cause Found & Fixed

**Investigation via Playwright MCP (browser_evaluate):**
- Navigated to http://localhost:8888/nbi_project_dashboard.html via Playwright-controlled Chrome
- Inspected computed styles of `.sidebar`, `.sidebar__nav`, `.sidebar__toggle` across all 8 themes
- Flex chain confirmed working: sidebar=1173px, nav=1141px, toggle=32px at viewport bottom

**Root cause:** CSS specificity conflict at line 309.
- Line 307: `.sidebar__toggle { ... color: var(--text-muted); border-top: 1px solid var(--border-default); background: var(--bg-raised); }` (specificity 0,1,0)
- Line 309: `button.sidebar__item, button.sidebar__toggle { ... color: inherit; background: none; border: none; }` (specificity 0,1,1 — WINS)
- Result: toggle gets transparent background, no border-top, and inherits parent color instead of using theme-aware `--text-muted`
- In dark themes: inherited color happens to contrast against dark sidebar bg → visible
- In light themes: inherited color doesn't contrast → invisible

**Fix applied:**
- Removed `button.sidebar__toggle` from combined reset selector at line 309 (now just `button.sidebar__item`)
- Added `font: inherit; width: 100%;` to `.sidebar__toggle` at line 307 (button resets the toggle still needs)
- Verified via browser_evaluate: all 8 themes now have `hasBg: true`, `hasBorder: true`, proper `--text-muted` color
- Visual verification: screenshots in both light and dark themes show visible toggle with border-top separator
- Functional verification: click toggle → sidebar collapses to 56px icon-only → click again → expands back
