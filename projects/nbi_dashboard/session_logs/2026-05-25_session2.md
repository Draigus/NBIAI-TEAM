## Flaky Test Investigation & Fix
- **Symptom:** Test failures varied wildly between runs (0, 7, 9, 80, 275 failures)
- **Root causes found (3):**
  1. Test pool in `tests/helpers/db.js` had no `error` event handler — when a PostgreSQL connection was terminated, the pool couldn't recover and all subsequent tests failed
  2. `truncate()` included `finance_entries` in TRUNCATE_TABLES but baseline-schema.sql only goes to migration 053 — if migration 054 failed to apply, TRUNCATE threw "relation does not exist" crashing every subsequent test
  3. Zombie node processes from previous test runs held idle connections that interfered with schema rebuilds
- **Fixes applied to `tests/helpers/db.js`:**
  - Added `keepAlive: true` + `keepAliveInitialDelayMillis: 10000` to pool config
  - Added `pool.on('error')` handler so killed connections don't poison the pool
  - `truncate()` now resolves TRUNCATE_TABLES against `pg_tables` on first call — missing tables are skipped instead of crashing
- **Result:** 600/600 passing, stable across 2 consecutive full-suite runs
