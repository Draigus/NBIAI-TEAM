### Architecture
- **Incremental build**: Phase 1 (dashboard + scanners), Phase 2 (nightly cron), Phase 3 (autonomous execution engine)
- **Approach A chosen**: Single route module `routes/command-centre.js` + one `cc_snapshots` table (JSONB, keyed by date)
- **Data freshness**: Cached with manual refresh button. Phase 2 cron auto-refreshes nightly.
- **File access**: Server gets full read access to NBI_TEAM root. Read-only, never writes.
- **Autonomy scope (Phase 3)**: Code changes only, always to worktree branch, never master. Glen reviews and merges.
- **Skill learnings**: Stored as `learnings.md` in each skill directory alongside SKILL.md.
