## Bug Fixes Completed

### Bug 817d084c — Date input year validation fires too early
- **Root cause:** Chrome `<input type="date">` fires `change` events with intermediate year values (e.g. "0002" when user types first digit of "2026"). The validation checked `yr < 1900` and showed error toast + called `renderContent()`, destroying the input and booting the user out.
- **Fix:** Added `if (yr < 1000) return;` to silently ignore partial years. Removed `renderContent()` from the error path in `updateTask`. Applied to both `updateTask` and `standupUpdateTask`.
- **Files:** nbi_project_dashboard.html (2 locations)

### Bug c0e9c8e8 — Est/spent hours label vs input mismatch
- **Root cause:** Detail panels showed aggregate hours (self + all descendants) in the summary label but the item's own hours (often 0 for parent items) in the editable inputs. Confusing mismatch.
- **Fix:** When an item has children, show aggregate hours in disabled inputs (matching existing pattern for auto-calculated dates). Leaf tasks keep editable inputs. Applied to both inline and overlay detail panels.
- **Files:** nbi_project_dashboard.html (2 locations — inline detail ~line 9514, overlay detail ~line 10277)

### Bug 7456e671 — Marking task done sets end date
- **Root cause:** New feature request — end date should auto-set when marking Done, clear when un-Done.
- **Fix:** In both `updateTask` and `standupUpdateTask`: when status changes to Done, set endDate to today. When status changes from Done, clear endDate. Undo correctly restores previous endDate. Parent date range propagation also fires on status-triggered endDate changes.
- **Files:** nbi_project_dashboard.html (2 functions — updateTask, standupUpdateTask + date propagation condition)

### Bug 1e6a4dfe — Ability to Delete Ticket Comments
- **Root cause:** Notes on task detail panels (both inline and overlay) had no delete buttons. Users could add notes but never remove them. The overlay panel's database-backed "Comments" already had delete buttons, but "Notes" (the section users interact with most) did not.
- **Fix:** Added `deleteNote(id, noteIndex)` function with confirmation prompt. Added red "delete" link next to the timestamp on every note in both panels. The function splices the note from `task.notes`, marks dirty, saves, and re-renders.
- **Files:** nbi_project_dashboard.html (3 locations — new function, inline panel notes, overlay panel notes)
- **Verified:** Playwright — delete button renders, note deletion works (1 → 0 notes), overlay re-renders correctly.
