### Task 7: M2 Shell Guard

**Started:** Session open
**Plan ref:** `docs/superpowers/plans/2026-06-14-rho-hardening.md` Task 7

**What it does:** PreToolUse hook for Bash/PowerShell that blocks recognised write primitives (>, >>, cp, mv, rm, Set-Content, Out-File, etc.) when they target governed paths (.claude/harness/config/, .claude/harness/lib/). Closes the bypass where shell commands could write to paths that write-guard.js blocks for Write/Edit tools.

**Approach:** TDD. Tests first, then implementation, then wire into settings.json.

**Progress:**

1. **Tests written** (42 cases across 16 groups):
   - Groups 1-6: BLOCK cases (redirects, PowerShell cmdlets, cp/mv/rm, node -e, python -c, tee)
   - Groups 7-9: ALLOW cases (no governed path, read-only ops, non-governed harness paths)
   - Groups 10-12: BLOCK (case-insensitive, backslash Windows, traversal attempts)
   - Group 13: Edge cases (empty, malformed, missing fields)
   - Group 14: ALLOW — quote-aware text arguments (codex exec prompts)
   - Group 15: BLOCK — cat redirect overrides read-only prefix
   - Group 16: BLOCK — chained commands (&&, ;)

2. **Implementation** — quote-aware parsing (major design evolution):
   - Initial naive approach (string contains write primitive + governed path) caused false positives on `codex exec` prompts discussing governed paths
   - Rewrote with three-phase quote-aware approach:
     - Phase 1: Redirect detection respecting quote state, checked BEFORE read-only prefix
     - Phase 2: Read-only prefix check (cat, head, grep, node --check, etc.)
     - Phase 3: Pipe segment analysis — splits by |, ;, && respecting quotes, checks first word of each segment against WRITE_COMMANDS set
   - Also handles node -e + fs write, python -c + open/write

**Verification run:** `node .claude/harness/tests/test-shell-guard.js` returned 76 passed, 0 failed. `Select-String "test\\("` returned 77 because it includes the helper function declaration.

**Review outcome:** Not a clean sign-off. Direct fixes for read-only prefix chaining, quote-in-path, backslash escape, and simple governed-path variable assignment are working. Remaining bypasses found:
- Wrapper handling only checks the second word, so `sudo -E cp`, `env FOO=bar cp`, and `xargs -I{} cp` are allowed.
- Shell exec detection misses `bash -lc`, allowing a governed write through a common login-shell form.
- Git write-subcommand detection only checks the second word, so `git -C . checkout HEAD -- .claude/harness/lib/write-guard.js` is allowed.
- Variable assignment detection only catches a full governed path in one assignment, so composed paths such as `p=.claude/harness; q=lib/evil.js; cp /tmp/x $p/$q` are allowed.
- Inline Node detection misses realistic `require('fs').writeFileSync(...)` shape because it only looks for `fs.writefilesync`.
- PowerShell aliases/native names such as `sc`, `ni`, `ri`, `del`, `copy`, and `move` are allowed against governed paths.
