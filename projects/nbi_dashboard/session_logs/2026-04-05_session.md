- New `downloadAttachment()` function: uses `authFetch` to get file as blob, creates a temporary object URL, triggers download via a programmatic `<a>` click
- Added delete button per attachment with `deleteAttachmentUI()` function

### 13:00 — Server crash protection added
**Problem:** Express 4 does not catch errors from `async (req, res) => {}` route handlers. Any unhandled DB error (missing table, bad query) caused `unhandledRejection` which killed the Node.js process. Server was crashing silently.
**Fix:**
- Installed `express-async-errors` (1 line: `require('express-async-errors')`) — patches Express to auto-forward async errors to the error handler
- Added `process.on('unhandledRejection')` as safety net — logs error but doesn't crash
- The existing `app.use((err, req, res, next) => {...})` error handler at line 1927 now correctly receives async errors and returns 500 JSON
