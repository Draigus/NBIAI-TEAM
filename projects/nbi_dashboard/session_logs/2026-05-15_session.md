## Session 3 — Critical Text-Saving Fix

**Context:** Stavros reported that text entered into fields (especially blocker info) is not saving. He entered data yesterday and it was lost. Glen flagged this as critical — users will abandon the tool if this happens.

### Root Cause Investigation (Systematic Debugging)

Traced the full save pipeline: user keystroke → onchange (blur) → updateTask() → markDirty() → save() → debounced syncToAPI() → POST /api/sync/changes.

**Found 3 confirmed failure modes:**

1. **PRIMARY: `onchange` on textareas never fires because DOM is destroyed before blur.** All text fields (description, collaborations, successFactor, blocker blockedOn) use `onchange` which only fires on blur. If `renderContent()` is called while the user is typing (clicking another task, switching tabs, filter change, etc.), the textarea is destroyed before blur fires. The typed text never reaches the in-memory task object. 50+ code paths call `renderContent()` directly — only the poll path had a guard (`_softReRender`).

2. **SECONDARY: `_serverUpdatedAt` stale after sync causes silent rejection.** After a successful sync, the client showed "Saved ✓" but never updated `_serverUpdatedAt`. If the user edited the same task again before the poll updated the timestamp, the next sync would be silently rejected by the server's conflict detection (`clientKnownAt < serverUpdatedAt` → `continue`).

3. **TERTIARY: No `beforeunload` handler.** If the user closed the tab while the 500ms sync debounce was pending, changes were lost. The previous "fix" (bug caf58563, marked resolved 2026-05-12) claimed to add one but it wasn't in the code.

### Fix Implemented

**Commit 1 (2365ebc):** Previous session's uncommitted work committed — lead filtering, SoW .txt upload, sort_order, no-cache headers.

**Changes (uncommitted, deployed to PM2):**

1. **`_liveWrite(taskId, field, value)` + `_liveWriteBlocker(taskId, subField, value)`** — Write to in-memory task on every keystroke via `oninput`. Debounced `save()` at 1.5s while typing.

2. **`_flushActiveEdits()`** — Force-saves any active input/textarea before DOM rebuild. Called at the top of `renderContent()` and `renderAll()`. Recursion-guarded.

3. **`oninput` handlers added to all 13 text fields:**
   - 4 title inputs (inline detail x2, overlay detail x2)
   - 4 description textareas (inline detail x2, overlay detail x2)
   - 2 collaborations textareas
   - 2 successFactor textareas
   - 1 blocker blockedOn textarea
   - Reporting drawer editBlock (risks, mitigations, documentation link, goals)

4. **`beforeunload` handler** — Flushes active edits, then fires keepalive fetch to /api/sync/changes with all dirty tasks.

5. **Server: `updatedTimestamps` in sync response** — UPDATE and INSERT now use `RETURNING updated_at`. Response includes `updatedTimestamps` map. Client updates `_serverUpdatedAt` from response, preventing stale conflict detection.

6. **Bug caf58563 reopened** with detailed root cause comment.
