### Feature: Internal Notifications Endpoint
**File:** `/d/OneDrive/Claude_code/NBIAI_TEAM/dashboard-server/server.js` (lines 939-966)

**Implementation:**
- Added `POST /api/internal/notifications` endpoint before `app.use(requireAuth)` middleware (mounted at line 939-966, auth wall at line 966)
- Authenticates via `x-nbi-internal-token` header matching `process.env.NEWS_INTERNAL_TOKEN`
- Supports two modes:
  1. `targetAdmins: true` — broadcasts to all admin users (filtered by `role = 'admin' AND is_active = true`)
  2. `username: "glen"` — sends to a specific user
- Uses existing `createNotification(username, type, title, message, link, dismissable)` helper (line 2257)
- Global JSON parser already in place at server startup (line 581), so inline middleware not needed

**Request schema:**
- `type` (required): notification type e.g. "info", "warning", "error"
- `title` (required): notification title
- `message` (optional): detailed message
- `link` (optional): notification link/action
- `dismissable` (optional, default true): whether user can dismiss
- `targetAdmins` (optional): if true, broadcast to all admins; requires either this or `username`
- `username` (optional): target username; required when targetAdmins not set

**Response schema:**
- Success (targetAdmins): `{"ok": true, "recipients": N}`
- Success (single user): `{"ok": true}`
- Error: `{"error": "message"}` with appropriate HTTP status (400, 401, 500)

**Error handling:**
- 401: Missing or invalid `x-nbi-internal-token`
- 400: Missing `type` or `title`, or missing both `targetAdmins` and `username`
- 500: Database error during notification creation
