## Technical Reference

### Paperclip API Patterns

```
Base URL: http://localhost:3100/api
Company ID: 359ab370-c36f-4558-a252-637255ad1a7b

# Goals
GET  /companies/{companyId}/goals         # list all goals
POST /companies/{companyId}/goals         # create goal {title, description, level, status, parentId, ownerAgentId}
PATCH /goals/{id}                         # update goal (partial)
DELETE /goals/{id}                        # delete goal (watch for FK constraints)

# Skills
GET  /companies/{companyId}/skills        # list registered skills
POST /companies/{companyId}/skills/import # import from local path {"source": "/path/to/skill-dir"}

# Issues
GET  /companies/{companyId}/issues        # list issues (supports ?assigneeAgentId= &status= filters)
POST /companies/{companyId}/issues        # create issue {title, description, status, priority, goalId, projectId, assigneeAgentId, parentId}

# Projects
GET  /companies/{companyId}/projects      # list projects
POST /companies/{companyId}/projects      # create project

# Agents
GET  /companies/{companyId}/agents        # list agents (returns all 32+)
PATCH /api/agents/{id}                    # update agent config
```
