## Immediate Next Task

**Write `projects/nbiai_app/app/src/routes/queue.ts`**

This is the core of the no-API architecture. Four endpoints:

### POST /queue/create
```
Body: { agentId: string, taskId: string }
```
1. Validate both UUIDs
2. Call `loadAgentContext(agentId, taskId)` from `../execution/context-loader.js`
3. Assemble `sessionPrompt` string from all context parts (see format below)
4. Create `claudeDesktopSession` record (status: `pending`, trigger: `manual`)
5. Update task: `status = 'queued'`, `queuedAt = now()`, `sessionId = newSession.id`, `sessionPrompt = assembled prompt`
6. Write JSON task file to `{REPO_PATH}/queue/inbox/{taskId}.json` (see queue_schema.md for format)
7. Return `{ task, sessionPrompt, sessionId }`

**Session prompt assembly format:**
```
{systemPrompt}

---

## Company Knowledge (Tier 1)

{tier1Knowledge}

---

## Role Knowledge (Tier 2)

{tier2Knowledge}

---

## Project Knowledge (Tier 3)

{tier3Knowledge}

---

{taskContext}
```

### GET /queue
```
Query: { status?: string, limit?: number, cursor?: string }
```
List tasks with queue-relevant statuses (queued/in_progress/review/done). Join agents, projects. Paginated.

### GET /queue/:taskId/prompt
Return the stored `sessionPrompt` for a task (for copy-to-clipboard in UI).
Auth required.

### POST /queue/results
```
Body: { taskId: string, sessionId?: string, status: 'done' | 'failed', output: string, notes?: string }
```
1. Update task: `status = status === 'done' ? 'review' : 'done'`, `output = output`, `completedAt = now()`
2. Update `claudeDesktopSession`: `status = 'completed' or 'failed'`, `completedAt = now()`, `notes`
3. Move queue file from `queue/active/{taskId}.json` to `queue/done/{taskId}.json` (or `failed/`)
4. Log activity
5. Return updated task
