# EAD Framework — Eliminate, Automate, Delegate

## Key Content

NBI's proprietary process optimisation methodology applied in strict order: 1) Eliminate (does this need to exist? kill criteria: if stopping for two weeks goes unnoticed, or only justified by "we've always done it"). 2) Automate (can a system do this without human judgement? good targets: data transformation, status aggregation, threshold monitoring, template generation, cross-system sync; poor targets: relationship-dependent decisions, ambiguous judgement, creative strategy, high-cost-of-error). 3) Delegate (match to appropriate capability tier: AI routine/autonomous, AI supervised/human-reviews, junior human, senior human, owner-only). Client engagement pattern: audit phase mapping all recurring processes, EAD pass on each, deliverable as process map showing eliminated/automated/delegated with justifications, priority on highest-frequency lowest-judgement processes first for visible ROI.

## Decisions / Insights

- Glen decided: order is strict (Eliminate BEFORE Automate) to prevent building systems for unnecessary processes
- Glen identified: key anti-pattern is "automating before eliminating" -- building sophistication around waste
- Glen decided: delegation tiers explicitly include AI agents at two levels (routine and supervised)
- Glen observed: "the cost of a wrong answer exceeds the cost of a slow answer" as the boundary for human retention
