# Glen's Quality Standard: CFO-Grade or Nothing

## Key Content

Glen's first live reaction to dashboard output set the permanent quality bar. His response to the initial Finances page: "That looks like absolute shit." He demanded CFO-grade quality, sent two reference dashboard images as design targets (Ajelix accounting, P&L tracking), and stated: "Nothing on this project is something you can do with minimal effort. It needs to be high quality." This established that pre-seeding real financial data was the correct approach over placeholder content, and that every view must be presentation-ready at first render.

## Decisions / Insights

- Glen judges output against professional tool standards, not "good for a side project"
- Reference images are how Glen communicates design intent -- he shows, not describes
- Real data trumps placeholder data in every case -- Glen confirmed pre-seeding NBI financials
- The quality bar applies universally, not just to client-facing work

## Context

First week of WorkSage development. This reaction shaped every subsequent design decision and established the pattern of Glen providing reference images when dissatisfied.

## Applicability

- Any visual deliverable for Glen must meet commercial SaaS quality standards
- When starting a new view/feature, find reference designs proactively rather than building from assumptions
- Pre-seed with real data whenever possible -- demos with fake data will be rejected
