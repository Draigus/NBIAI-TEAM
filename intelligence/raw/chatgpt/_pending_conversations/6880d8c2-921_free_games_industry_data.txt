As an expert data scientist and video game forecaster, your task is to **design an agentic, coded forecast model simulator for video games**. This system will forecast key performance metrics for games across **mobile, Telegram, PC (Steam), and console platforms**, covering all genres. The model should be *agentic* – meaning it can autonomously gather data from web sources and APIs – and incorporate advanced time series forecasting techniques. The end product is a **reusable, resilient forecasting tool** with a **user-friendly (lovable) GUI** that allows interactive scenario analysis. Below is a detailed prompt and specification outlining the requirements, features, and design considerations for this simulator:

## Overview and Objectives

- **Scope**: Build a forecast simulator that can handle **any video game title or genre**, including mobile apps, social/Telegram games, PC (Steam) games, and console titles. It should cover *free-to-play, premium/box, live service,* or hybrid business models.
- **Metrics Forecasted**: The simulator will project a comprehensive set of game KPIs (Key Performance Indicators) and metrics, including but not limited to:  
  - **User Engagement**: Daily Active Users (DAU), Monthly Active Users (MAU), Session Count, Average Session Length, Average Playtime per week/month/year, Retention rates (D1, D7, D30, etc.)【14†L129-L134】, Churn rate, Concurrent Users (CCU).  
  - **Monetization**: Average Revenue Per User (ARPU), Average Revenue Per Paying User (ARPPU), Average Revenue Per Daily Active User (ARPDAU), Lifetime Value (LTV), Conversion rate (conversion of non-paying to paying users), Revenue (total, by source), etc.【11†L173-L182】【11†L197-L204】.  
  - **Acquisition & Marketing**: Installs (organic vs paid), User Acquisition (UA) metrics, Retention (as above), Acquisition cost (CAC), Cost Per Install (CPI) / Cost Per Click (CPC), Return on Ad Spend (ROAS), Click-through rate.  
- **Time Frame**: The user can select forecast horizons of **1, 2, 3, 4, or 5 years** into the future. The model should offer **granular outputs** at daily, monthly, and yearly levels (e.g. daily forecasts aggregated into monthly/yearly summaries). This supports both short-term tactical forecasts and long-range planning.
- **Goal**: Enable users to **simulate the future performance** of a specific game or an entire genre segment by leveraging genre benchmarks and historical data. The simulator should help identify potential growth, peak performance, and post-launch trends, and also highlight challenges (e.g. retention issues or monetization shortfalls) in a **written summary report** for each forecast.

## Data Gathering and Baseline Initialization

- **Automated Data Collection (Agentic Behavior)**: The model will employ an **agentic approach** to gather relevant historical data and benchmarks. It should automatically research and pull in data from **all available web sources** (public APIs, databases, or websites) for the game’s genre and platform:
  - *Examples of Data Sources*: industry databases like data.ai (App Annie) for mobile metrics, Sensor Tower for mobile revenue and download estimates, Steam Charts for PC concurrency and player counts, console game sales databases, and game analytics benchmarks (e.g., GameAnalytics reports)【14†L179-L185】. If direct API access is not available, the agent can perform web searches to find published stats or reports on similar games.
  - **Genre Baseline Data**: For a given title, the system first identifies its genre (or multiple genres/subgenres). It then gathers **genre-wide baseline metrics** – e.g., average D1/D7 retention, typical ARPU, typical DAU curves over time – by analyzing data from other games in that genre【14†L125-L133】. This establishes a **baseline performance curve** for each metric, which the forecast will use as a starting point. The agent should scour the web for genre benchmarks (e.g., “RPG mobile game average 30-day retention” or “FPS console average ARPU”) and compile these figures.
  - **User Input & Data Upload**: The system will allow the user to provide data as well. If the user has proprietary metrics or a dataset (e.g., CSV/Excel of historical performance, or known genre averages), they can upload it. The model will then incorporate this data to improve accuracy. *For example*, a user could upload baseline KPIs for the “Battle Royale” genre gleaned from market research, and the model should then use those as priors for forecasting a new Battle Royale title.
- **Data Storage and Updating**: All gathered data (both from web and user uploads) should be stored or cached so that:
  - The model can reuse genre stats for future forecasts (building a growing internal database of genre benchmarks).
  - The system can be updated with new data over time (e.g., ingest quarterly industry reports to refresh the baseline).
- **Quality and Validation of Data**: The agent must verify the credibility of sources and data consistency. For instance, if multiple sources report different DAU numbers for similar games, the model might take an average or prioritize the most reputable source. **Outlier detection** or anomaly checks should be in place to avoid using any obviously incorrect data (e.g., a typo on a website reporting an impossibly high retention >100% would be ignored or corrected).

## Forecasting Engine and Methodology

- **Advanced Time Series Modeling**: Utilize a **state-of-the-art multivariate time series forecasting model** to project all metrics forward in time simultaneously. The model should capture the interdependencies between metrics – e.g., how DAU trends might affect revenue or how installs translate to active users and then to paying users. Two recommended approaches to consider:
  - **Vector Autoregression (VAR)**: A classical multivariate approach where each metric’s future values are a function of past values of *all* metrics in the system【16†L84-L91】. VAR is useful for capturing relationships (e.g., how changes in retention or installs impact future DAU) and provides a statistically sound baseline. However, VAR assumes linear relationships and may struggle with complex patterns or non-stationary trends without adjustments.
  - **Transformer-based Time Series Model (TimeGPT)**: Leverage a modern deep learning approach, such as **Nixtla’s TimeGPT** – a pre-trained generative transformer model for time series forecasting【17†L148-L155】. TimeGPT is a **foundation model trained on over 100 billion data points**, capable of accurate zero-shot forecasts across diverse domains【17†L148-L155】. It consistently outperforms traditional models like ARIMA or basic machine learning models in capturing complex patterns in data【9†L139-L147】【10†L569-L577】. Using TimeGPT (or a similar large-scale model) can allow the simulator to handle nonlinear trends, seasonality, and unusual spikes/dips with greater accuracy. It also reduces the need for extensive parameter tuning, as it works out-of-the-box with a simple API call【9†L125-L133】【9†L139-L147】.
  - **Model Selection**: Given the requirement to run on a personal PC for now, the solution should be **modular**. We can start with a lighter-weight implementation (VAR or a smaller-scale model) for local testing. Ensure that the architecture allows swapping in the more powerful model (like TimeGPT via API or a local neural model) when scaling up or when running in a cloud environment. The prompt should emphasize choosing the **“best rated, least expensive”** option by default – likely an open-source library or model that balances performance and cost. For example, using Facebook Prophet or an open-source neural forecasting library as a fallback could be cost-effective, while later integrating TimeGPT’s API for higher accuracy when budget allows.
- **Exogenous and Causal Factors**: The forecasting model should allow incorporating external factors (exogenous variables) such as:
  - Major content updates or expansions (which often cause surges in engagement).
  - Marketing campaigns or ad spend fluctuations.
  - Seasonality (holidays, weekends vs weekdays patterns, regional trends).
  - Platform changes (e.g., a game launching on a new platform mid-forecast).
  The model/agent can prompt the user for these events. For instance, *“Do you plan any major updates or marketing pushes during the forecast period? If so, specify the expected dates and impact.”* These inputs can adjust the forecast (e.g., a temporary DAU boost during a campaign, or higher new installs around a known holiday season for gaming).
- **Autonomous Validation (Mitigating Hallucinations)**: To ensure reliability, the system should **validate its outputs**:
  - After generating a forecast, the simulator will **cross-check the results against baseline expectations**. It knows the genre averages and competitor benchmarks it gathered; thus if the forecasted metrics diverge wildly from any realistic range, it flags this. For example, if the genre’s typical 1-year retention is under 40% and the model somehow predicts 70% for the new game, it should recognize this as likely erroneous. The agent can then either adjust the forecast downwards or ask the user for confirmation (*“The predicted retention is much higher than genre norms; is there a specific reason your game would significantly outperform the market?”*).
  - Leverage the model’s anomaly detection capabilities. Notably, **TimeGPT has built-in anomaly detection** features【7†L109-L117】. The system can run an anomaly detection pass on the forecasted series to spot any unusual patterns, further reducing the chance of spurious outputs.
  - Perform **multiple simulations/iterations** to see if consistent results are obtained. The prompt might instruct the AI to run the forecast process several times (with slight variations or different random seeds if applicable) and then average or choose the most plausible outcome, ensuring stability.
- **Forecast Outputs**: For each metric, the model should provide:
  - **Point forecasts** (the expected value each day/month).
  - **Confidence intervals or error bounds** (e.g., 95% confidence interval bands for each metric’s trajectory), so the user understands the uncertainty. This can be derived from the model (some models provide prediction intervals) or via ensemble/bootstrapping methods.
  - **Relevant comparisons** to baseline: e.g., “Forecasted DAU after 1 year is 500k, which is ~1.2x the current genre average for top games in this genre” or “Forecasted ARPU is $0.30, slightly below the genre benchmark of $0.35” – these insights help contextualize the numbers.

  4. **Historical Data**: If forecasting an existing game, the user can input historical metrics. For a new game, this step may be skipped or replaced with “use genre averages as a starting point.” The UI should allow either manual entry of known KPIs or uploading a file (CSV/Excel) with historical time series.
  5. **Target Regions (optional)**: If regional breakdown is needed, let the user select key markets (e.g., North America, Europe, Asia).   7. **Custom Adjustments & Assumptions**: Present a set of advanced options where the user can tweak assumptions:

     - Modify growth or decay rates for metrics if known (e.g., “Our marketing plan will boost new installs by 20% on month 2”).
     - Input known events (as discussed in exogenous factors: content updates, ad campaigns, etc., with expected effects).
     - Toggle on/off the inclusion of certain metrics if not applicable (for instance, a console game might not have “installs” in the same sense as a mobile app, or a single-player premium game might not track DAU as closely post-launch).

  - Each chart is interactive: the user can hover to see exact values on specific dates, and toggle series on/off (for example, show DAU vs MAU on the same graph, or compare organic vs paid installs).
  - The top of the dashboard will have **filter controls** (e.g., checkboxes or dropdowns) to refine the view:
    - **Time filter**: Zoom in to a specific date range or toggle between viewing daily vs monthly data.

    - **Metric selector**: Ability to highlight one metric’s chart at a time or overlay multiple metrics for comparison.
    - **Region filter** (if regional data provided): select which region’s data to display.
  - Include **benchmark overlays**: An optional feature is to overlay genre benchmark lines on the charts. For example, on the DAU chart, show the **genre average DAU trajectory** (for a game of this size/age) as a dashed line, so the user can immediately see if the forecast is predicting above or below market norms【14†L179-L185】.
  - **Customization and Notes**: The UI will let the user add annotations or notes on the charts. Perhaps a right-click or a button “Add Note” where the user can drop a comment like “Spike due to expansion release here” on a specific date. These notes should be saved with the forecast scenario (so when the user loads it later, the notes appear).
- **Output Reports**:
  - The user should be able to **export the forecast data** easily. Provide a one-click **Excel export** that contains all the raw forecast numbers (and possibly the input assumptions on a separate sheet). The Excel should be well-formatted, with tabs for each metric or a clear tabular presentation of metrics by day/month.
  - Additionally, allow export of the visualizations (e.g., a button to download a PNG image of a chart, or a PDF report containing all charts).
  - **Written Summary**: A key feature is an auto-generated **written analysis** of the forecast. After running the simulation, the system will produce a report summarizing:

```python src/data_ingest.py
"""
Data-ingestion utilities for the Playsage forecasting engine.

Responsibilities
----------------
1. Pull small starter datasets from free public APIs (Steam, IGDB, Apple, AppBrain).
2. Accept user-supplied CSV / Excel files and normalise column names.
3. Persist tidy tables into the local SQLite database defined in config.DB_PATH.

All external API calls are deliberately minimal so the file remains runnable
without paid keys.  Extend the stub methods as you add real credentials.
"""

from __future__ import annotations

import json
import sqlite3
from pathlib import Path
from typing import Iterable, Literal

import pandas as pd
import requests

from .config import DATA_DIR, DB_PATH, ensure_key

# ──────────────────────────────────────────────────────────────
# Helpers
# ──────────────────────────────────────────────────────────────
def _init_db() -> sqlite3.Connection:
    """Return a SQLite connection; create DB file if missing."""
    DB_PATH.parent.mkdir(exist_ok=True)
    conn = sqlite3.connect(DB_PATH)
    conn.execute("PRAGMA journal_mode=WAL;")
    return conn


def _to_sql(df: pd.DataFrame, table: str, if_exists: Literal["replace", "append"] = "append") -> None:
    """Write DataFrame to SQLite, ensuring column names are snake_case."""
    df.columns = [c.strip().lower().replace(" ", "_") for c in df.columns]
    with _init_db() as conn:
        df.to_sql(table, conn, if_exists=if_exists, index=False)


# ──────────────────────────────────────────────────────────────
# Steam
# ──────────────────────────────────────────────────────────────
def fetch_steam_app_details(app_id: int) -> pd.DataFrame:
    """
    Pull basic app metadata from the public Steam Store API.

    No key required.  Returns minimal fields to keep token count low.
    """
    url = f"https://store.steampowered.com/api/appdetails?appids={app_id}&cc=us&l=en"
    resp = requests.get(url, timeout=10)
    resp.raise_for_status()

    data = resp.json()
    if not data[str(app_id)]["success"]:
        raise ValueError(f"Steam appid {app_id} not found.")

    info = data[str(app_id)]["data"]
    record = {
        "appid": app_id,
        "name": info.get("name"),
        "release_date": info.get("release_date", {}).get("date"),
        "price_usd": (
            info.get("price_overview", {}).get("final", 0) / 100
            if info.get("is_free") is False
            else 0
        ),
        "is_free": info.get("is_free"),
        "genres": ", ".join(g["description"] for g in info.get("genres", [])),
    }
    return pd.DataFrame([record])


# ──────────────────────────────────────────────────────────────

def _igdb_request(query: str) -> list[dict]:
    """
    Very small wrapper for IGDB 'games' endpoint using Twitch credentials.

    Replace the placeholders with real OAuth process if you have keys.
    """
    client_id = ensure_key("IGDB")  # raises if placeholder
    token = ensure_key("STEAM")     # re-use STEAM placeholder for brevity

    headers = {
        "Client-ID": client_id,
        "Authorization": f"Bearer {token}",
    }
    url = "https://api.igdb.com/v4/games"
    resp = requests.post(url, headers=headers, data=query, timeout=10)
    resp.raise_for_status()
    return resp.json()

        "Monthly Active Users": "mau",
```

```python src/benchmark_agent.py
def get_genre_benchmark(
    metric: str,
    genre: str,
    platform: str | None = None,
) -> float | None:
    """
    Return the benchmark value for *metric* in *genre* (and optionally platform).

    row = df.loc[mask]
    return None if row.empty else float(row.iloc[0]["value"])

    csv_path = DATA_DIR / "genre_benchmarks.csv"
```

```python src/model_var.py
"""
VAR forecasting model for the Playsage engine.

Only the essentials are included so it runs quickly on a laptop.
Extend with grid-search or exogenous variables once the pipeline works.
"""

from __future__ import annotations

from typing import Sequence

import numpy as np
import pandas as pd
from statsmodels.tsa.api import VAR

DEFAULT_LAG = 12  # months


def train_var(
    df: pd.DataFrame,
    target_cols: Sequence[str],
    max_lag: int = DEFAULT_LAG,
) -> VAR:
    """
    Fit a Vector Auto Regression model on *target_cols*.

    Parameters
    ----------
    df
        DataFrame indexed by period (datetime) and containing numeric columns.
    target_cols
        List of column names to include in the VAR system.
    max_lag
        Lag order to fit.  Auto-selects if <= 0.

    Returns
    -------
    statsmodels.tsa.api.VAR (fitted)
    """
    model = VAR(df[target_cols])

    if max_lag <= 0:
        # Select lag that minimises AIC up to 15
        results = model.select_order(15)
        max_lag = int(results.aic)

    results = model.fit(max_lag)
    return results


def forecast_var(
    fitted: VAR,
    periods: int,
    alpha: float = 0.05,
) -> pd.DataFrame:
    """
    Produce a point forecast and confidence intervals.

    Returns DataFrame with columns:
        <metric>_forecast, <metric>_lower, <metric>_upper
    """
    fc, lower, upper = fitted.forecast_interval(
        y=fitted.y,
        steps=periods,
        alpha=alpha,
    )
    columns = fitted.names
    fc_df = pd.DataFrame(fc, columns=[f"{c}_forecast" for c in columns])
    fc_df[[f"{c}_lower" for c in columns]] = lower
    fc_df[[f"{c}_upper" for c in columns]] = upper
    fc_df.index = pd.RangeIndex(start=1, stop=periods + 1, step=1)
    return fc_df
```

```python src/model_var.py
    """
    Produce point forecasts plus confidence intervals (1-alpha).

    Returns
    -------
    DataFrame with columns <metric>_forecast, <metric>_lower, <metric>_upper
    and an index 1…periods.
    """
    # statsmodels ≥0.14 requires the history array as first positional arg
    history = fitted.model.endog
    fc, lower, upper = fitted.forecast_interval(
        history,
        steps=periods,
        alpha=alpha,
    )

    cols = fitted.names
    fc_df = pd.DataFrame(fc, columns=[f"{c}_forecast" for c in cols])
    fc_df[[f"{c}_lower" for c in cols]] = lower
    fc_df[[f"{c}_upper" for c in cols]] = upper
    fc_df.index = pd.RangeIndex(1, periods + 1)
    return fc_df
```

```python src/validate.py
"""
Forecast-validation helpers.

• Compares forecast metrics to genre benchmarks.
• Flags any metric whose point estimate is more than N standard deviations
  away from the benchmark mean.
"""

from __future__ import annotations

import math
from typing import Sequence

import pandas as pd

from .benchmark_agent import get_genre_benchmark


def flag_outliers(
    forecasts: pd.DataFrame,
    metric_names: Sequence[str],
    genre: str,
    platform: str | None = None,
    z_threshold: float = 2.0,
) -> list[str]:
    """
    Return a list of warnings for metrics that deviate strongly
    from genre averages.

    Parameters
    ----------
    forecasts   DataFrame returned by model_var or model_timegpt
    metric_names
        base names (e.g. ["dau", "arpu"]) – function will look for
        '<metric>_forecast' columns.
    genre       genre label used in benchmark look-ups
    platform    optional platform filter
    z_threshold how many std-devs constitute an outlier
    """
    warnings: list[str] = []

    for m in metric_names:
        col_name = f"{m}_forecast"
        if col_name not in forecasts.columns:
            continue

        # simple: use first forecast point
        fc_val = float(forecasts.iloc[0][col_name])
        bench = get_genre_benchmark(m, genre, platform)

        if bench is None or math.isnan(fc_val):
            continue

        # crude stdev estimate: 15 % of benchmark
        stdev = 0.15 * bench
        z = (fc_val - bench) / stdev if stdev else 0.0

        if abs(z) >= z_threshold:
            warnings.append(
                f"{m.upper()} forecast {fc_val:,.2f} is {z:+.1f}σ vs "
                f"genre benchmark {bench:,.2f}"
            )

    return warnings


# ──────────────────────────────────────────────────────────────
# Smoke-test
# ──────────────────────────────────────────────────────────────
if __name__ == "__main__":
    # tiny DataFrame mimicking a forecast
    df = pd.DataFrame(
        {
            "dau_forecast": [50_000],
            "arpu_forecast": [0.80],
        }
    )
    msgs = flag_outliers(
        df,
        metric_names=["dau", "arpu"],
        genre="RPG",
        platform="mobile",
    )
    print("Warnings:", msgs or "none")
```

```python src/cli.py
"""
forecast_engine CLI entry-point.

Usage
-----
python -m src.cli new
"""

from __future__ import annotations

import sys
from datetime import datetime
from pathlib import Path

import pandas as pd
from prompt_toolkit import prompt
from prompt_toolkit.completion import WordCompleter
from rich import print

from .benchmark_agent import export_csv
from .data_ingest import ingest
from .model_timegpt import train_and_forecast
from .validate import flag_outliers

GENRES = ["RPG", "FPS", "Puzzle", "Strategy", "Sports"]
PLATFORMS = ["pc", "console", "mobile", "telegram"]


def ask(question: str, completer: WordCompleter | None = None) -> str:
    """Simple wrapper over prompt_toolkit.prompt."""
    return prompt(f"[cyan]{question}[/] ", completer=completer).strip()


def run_wizard() -> dict:
    """Collect forecast parameters interactively."""
    params: dict = {}

    params["game_name"] = ask("Game title?")
    genre = ask(
        "Primary genre?",
        completer=WordCompleter(GENRES, ignore_case=True),
    )
    params["genre"] = genre or "RPG"

    platform = ask(
        "Platform?",
        completer=WordCompleter(PLATFORMS, ignore_case=True),
    )
    params["platform"] = platform or "pc"

    horizon = ask("Forecast horizon (years, 1-5)?")
    params["horizon_years"] = int(horizon or "3")

    steam_id = ask("Steam appid (or blank if none)?")
    params["steam_id"] = int(steam_id) if steam_id.isdigit() else None

    return params


def main() -> None:
    if len(sys.argv) < 2 or sys.argv[1] != "new":
        print("[bold red]Usage:[/] python -m src.cli new")
        sys.exit(1)

    params = run_wizard()
    print(f"[green]Parameters:[/] {params}")

    # 1. Data ingest
    ingest(
        app_id=params["steam_id"],
        game_name=params["game_name"],
        user_file=None,
    )

    # 2. Build a tiny synthetic history just so the demo runs
    periods_hist = 24  # months
    dates = pd.date_range("2023-01-31", periods=periods_hist, freq="M")
    df_hist = pd.DataFrame(
        {
            "dau": range(1_000, 1_000 + periods_hist * 50, 50),
            "arpu": [0.25 + i * 0.002 for i in range(periods_hist)],
        },
        index=dates,
    )

    # 3. Forecast
    periods_fc = params["horizon_years"] * 12
    fc = train_and_forecast(
        df_hist,
        target_cols=["dau", "arpu"],
        periods=periods_fc,
    )
    print("\n[bold]Head of forecast:[/]")
    print(fc.head())

    # 4. Validate
    warns = flag_outliers(
        fc,
        metric_names=["dau", "arpu"],
        genre=params["genre"],
        platform=params["platform"],
    )
    if warns:
        print("\n[yellow]Benchmark warnings:[/]")
        for w in warns:
            print(f" • {w}")
    else:
        print("\n[green]No benchmark warnings.[/]")

    # 5. Save to Excel
    out_dir = Path("output") / params["game_name"].replace(" ", "_")
    out_dir.mkdir(parents=True, exist_ok=True)
    excel_path = out_dir / f"forecast_{datetime.now():%Y%m%d_%H%M}.xlsx"
    fc.to_excel(excel_path, index_label="step")
    print(f"\n[cyan]Forecast saved to:[/] {excel_path}")

    # 6. Export benchmarks CSV for convenience
    csv_path = export_csv()
    print(f"[cyan]Benchmarks CSV saved to:[/] {csv_path}")


if __name__ == "__main__":
    main()
```

```python src/data_ingest.py
"""
Loads KPI CSV/Excel and (optionally) enriches with Steam / IGDB calls.
"""

import sqlite3
from pathlib import Path
from typing import Optional

import pandas as pd
import requests

from .config import DB_PATH, API_KEYS

CORE_COLS = [
    "dau",
    "mau",
    "arpu",
    "arppu",
    "arpdau",
    "ltv",
    "retention_d1",
    "retention_d7",
    "retention_d30",
    "installs",
    "cpi",
    "roas",
    "cac",
    "ccu",
    "sessions",
    "organic_installs",
    "paid_installs",
    "playtime",
    "ctr",
    "cpc",
]


def _db() -> sqlite3.Connection:
    return sqlite3.connect(DB_PATH)


def load_local(file: Path) -> pd.DataFrame:
    df = pd.read_csv(file) if file.suffix == ".csv" else pd.read_excel(file)
    df["date"] = pd.to_datetime(df["date"])
    df = df.set_index("date").sort_index()
    return df


def fetch_steam_appdetails(appid: int) -> Optional[pd.DataFrame]:
    key = API_KEYS.get("STEAM")
    if not key or "YOUR_" in key:
        return None
    url = f"https://store.steampowered.com/api/appdetails?appids={appid}&cc=us&l=en"
    r = requests.get(url, timeout=10).json()
    data = r[str(appid)]["data"]
    return pd.DataFrame(
        {
            "name": [data["name"]],
            "price_usd": [data["price_overview"]["final"] / 100 if not data["is_free"] else 0],
            "release_date": [data["release_date"]["date"]],
        }
    )


def ingest(file: Path, appid: int | None = None) -> pd.DataFrame:
    df = load_local(file)

    if appid:
        meta = fetch_steam_appdetails(appid)
        if meta is not None:
            meta.to_sql("steam_meta", _db(), if_exists="append", index=False)

    df.to_sql("kpis", _db(), if_exists="replace")
    return df
```

```python src/benchmark_agent.py
"""Starter genre benchmarks—replace with real dataset when available."""

import pandas as pd

_BENCH = pd.DataFrame(
    [
        ("RPG", "pc", "arpu", 0.35, 0.05),
        ("RPG", "pc", "dau", 50000, 8000),
        ("FPS", "console", "arpu", 0.55, 0.07),
```

```python src/model_var.py
"""Multivariate VAR helper (auto-lag via AIC)."""

from typing import Sequence

import pandas as pd
from statsmodels.tsa.api import VAR


def train(df: pd.DataFrame, cols: Sequence[str]) -> VAR:
    model = VAR(df[cols])
    lag = int(model.select_order(15).aic)
    return model.fit(lag)


def forecast(fitted: VAR, steps: int) -> pd.DataFrame:
    fc = fitted.forecast(fitted.y, steps)
    return pd.DataFrame(fc, columns=[f"{c}_forecast" for c in fitted.names])
```

```python src/model_timegpt.py
"""Fallbacks to VAR if no TIMEGPT_API_KEY."""

import os
from typing import Sequence

import pandas as pd

from .config import API_KEYS
from .model_var import forecast as var_forecast, train as var_train

try:
    from nixtlats import TimeGPT
except ImportError:  # no nixtlats? fall back silently
    TimeGPT = None


def train_and_forecast(df: pd.DataFrame, cols: Sequence[str], periods: int) -> pd.DataFrame:
    key = API_KEYS.get("TIMEGPT", "")
    if TimeGPT is None or "YOUR_" in key:
        fitted = var_train(df, cols)
        return var_forecast(fitted, periods)

    tgpt = TimeGPT(api_key=key)
    parts = []
    for c in cols:
        yhat = tgpt.forecast(series=df[c].tolist(), horizon=periods, freq="D")
        parts.append(pd.Series(yhat, name=f"{c}_forecast"))
    return pd.concat(parts, axis=1)
```

```python src/validate.py
"""Flag forecasts >2 σ from genre means."""

from typing import Sequence

import pandas as pd

from .benchmark_agent import lookup


def flags(fc: pd.DataFrame, metrics: Sequence[str], genre: str, platform: str) -> list[str]:
    warn = []
    for m in metrics:
        val = fc.iloc[0][f"{m}_forecast"]
        mean, std = lookup(m, genre, platform)
        if mean is None:
            continue
        if abs(val - mean) > 2 * std:
            warn.append(f"{m.upper()} = {val:.2f} > 2σ away from genre ({mean:.2f})")
    return warn
```

```python src/narrative.py
"""Generate strengths/risks summary via GPT-4 (LangChain)."""

from langchain_openai import ChatOpenAI
from langchain.prompts import PromptTemplate

TPL = PromptTemplate.from_template(
    "Write a 300-word max executive summary of this forecast. "
    "List 3 strengths and 3 risks. Mention any metric far from benchmarks: {warn}."
)


def make_text(warn: list[str]) -> str:
    llm = ChatOpenAI(model_name="gpt-4o", temperature=0.3)
    return llm.invoke(TPL.format(warn=", ".join(warn)))
```
