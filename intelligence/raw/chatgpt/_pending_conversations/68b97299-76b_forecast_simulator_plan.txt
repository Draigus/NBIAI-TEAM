## 6.2 Retention_Curve
- A1 “Day”, B1 “Retention”
- A2 = 0, fill down to A{Horizon_Days+1}
- In a helper block (say E2:H10) enter anchor days and values for both Benchmark and Target:
  - For Benchmark: E2=0, F2=1.0; E3=1, F3=Retention_D1_Benchmark; E4=3, F4=AVERAGE(Retention_D1_Benchmark, Retention_D7_Benchmark); E5=7, F5=Retention_D7_Benchmark; E6=14, F6=MAX(Retention_D7_Benchmark*0.6, Retention_D28_Benchmark*1.2); E7=28, F7=Retention_D28_Benchmark; E8=60, F8=Retention_D28_Benchmark*0.55; E9=90, F9=Retention_D28_Benchmark*0.40
  - For Target: G2=0, H2=1.0; G3=1, H3=Retention_D1_Target; G4=3, H4=AVERAGE(Retention_D1_Target, Retention_D7_Target); G5=7, H5=Retention_D7_Target; G6=14, H6=MAX(Retention_D7_Target*0.6, Retention_D28_Target*1.2); G7=28, H7=Retention_D28_Target; G8=60, H8=Retention_D28_Target*0.55; G9=90, H9=Retention_D28_Target*0.40
- In B2 use a piecewise interpolation that switches sets based on Benchmark_vs_Target. Easiest path in Excel 365 is to spill a UDF-style LET with XMATCH and XLOOKUP. A practical version row by row is:
  ```
  =LET(
    d, A2,
    mode, Benchmark_vs_Target,
    days, IF(mode="Benchmark", $E$2:$E$9, $G$2:$G$9),
    vals, IF(mode="Benchmark", $F$2:$F$9, $H$2:$H$9),
    lastDay, 90,
    IF(d<=lastDay,
       LET(
         k, XMATCH(d, days, 1, 1),
         d0, INDEX(days, k), d1, INDEX(days, k+1),
         v0, INDEX(vals, k), v1, INDEX(vals, k+1),
         IF(d1=d0, v0,
            EXP( LN(v0) + (LN(v1)-LN(v0)) * (d-d0)/(d1-d0) )
         )
       ),
       LET(
         v90, INDEX(vals, XMATCH(lastDay, days, 1, 1)+1),
         EXP( LN(v90) - LN(2)*(d-lastDay)/Retention_Tail_HalfLife_Days )
       )
    )
  )
  ```
Fill to day 180. This gives the daily retention fraction.

## 6.5 Cohorts
- Build a daily installs table by joining Acq_Channels to CPI:
  - Inst_iOS = Spend_iOS / CPI_iOS
  - Inst_Android = Spend_Android / CPI_Android
  - Inst_Telegram = Spend_Telegram / CPI_Telegram
  - Paid_Total = sum of the three
  - Inst_Organic = Paid_Total * Organic_Install_Multiplier
  - Inst_Viral = Paid_Total * K_Factor for day greater than 0, else 0
  - Installs_Total = Paid_Total + Organic + Viral
- DAU via diagonal cohort sums. If you use Excel 365 you can compute DAU with a single spilled formula in Forecast_Daily (see below). If you prefer a grid, create columns D0 to D180 and in J2 use:
  ```
  = $G2 * INDEX(Retention_Curve!$B:$B, COLUMN()-COLUMN($I$1)+1)
  ```
  then DAU for day t is the sum of column Dt on or after row 2 down to row t+2.

## 6.6 Monetization
- ARPDAU path
  - ARPDAU_Base = IF(Benchmark_vs_Target="Benchmark", ARPDAU_Base_Benchmark, ARPDAU_Base_Target)
  - Apply daily multipliers from Content_Roadmap and weekend seasonality.
- Payer path
  - Cumulative payer conversion curve to D90 as a sigmoid:
    - CumConv(d) = Payer_Conv_D90_Target*(1/(1+EXP(−k*(d−x0)))) with x0 around 35 and k around 0.08
  - Daily new payers = Installs_Total * [CumConv(d) − CumConv(d−1)]
  - Revenue_Payer = DailyPayers * ARPPU_Target
- Blend
  - Revenue_Total = α*Revenue_ARPDAU_Path + (1−α)*Revenue_Payer_Path where α is a Control Panel slider.

# 8) How to use the simulator
1) Set Start_Date and Horizon_Days.  
2) Choose Benchmark vs Target. Benchmark loads industry medians (conservative), Target loads your QnA values.  
3) Fill UA spend per channel and CPI. If you scale spend, use UA_Spend_Decay_Percent or set a weekly pattern.  
4) Tune retention anchors and tail half-life. Keep D1, D7, D28 realistic relative to each other.  
5) Pick monetization approach. If early, favor payer-path. When live, increase ARPDAU blend.  
6) Define live-ops cadence on Content_Roadmap or let the Control Panel generate defaults.  
7) Review Daily and Weekly tabs. Compare ARPDAU Effective against your base to see if events are pulling their weight.  
8) Measure CCU estimate to inform server planning.  
9) Use the Competitor sheet to plug in new numbers as you get data.ai or Sensor Tower pulls. The model will not break if you leave them blank.

# 9) Core formulas and where they live
- Installs by channel: Spend ÷ CPI (Acq_Channels)  
- Total installs: Paid + Organic + Viral (Cohorts)  
- Daily cohort DAU contribution: Cohort_Size × Retention(d) (Cohorts)  
- DAU: diagonal sum of cohorts (Forecast_Daily)  
- Payers per day: Installs × [CumConv(d) − CumConv(d−1)] (Monetization)  
- Payer revenue: Daily_Payers × ARPPU (Monetization)  
- ARPDAU revenue: DAU × ARPDAU_Base × live-ops multipliers (Monetization)  
- Battle pass revenue: ActiveUsers × Attach/28 × Price, and Commander from Month 3 (BattlePass)  
- ARPDAU Effective: Revenue_Total ÷ DAU (Forecast_Daily)  
- CCU estimate: DAU × Sessions_per_User × Session_Length_min ÷ 1440 (Forecast_Daily)

def write_cp_value(row_idx, key, val, note):
    cp.write(row_idx, 0, key)
    if isinstance(val, str) and len(val) == 10 and val[4] == "-" and val[7] == "-":
        y, m, d = map(int, val.split("-"))
        cp.write_datetime(row_idx, 1, date(y, m, d), fmt_date)
    else:
        if isinstance(val, (int, float)):
            if any(s in key for s in ["Retention_", "Mix", "Lift", "Attach", "K_Factor", "Decay", "_Share"]):
                cp.write_number(row_idx, 1, val, fmt_pct)
            elif any(s in key for s in ["Price", "ARPDAU", "ARPPU"]):
                cp.write_number(row_idx, 1, val, fmt_cur)
            else:
                cp.write_number(row_idx, 1, val)
        else:
            cp.write(row_idx, 1, val)
    cp.write(row_idx, 2, note, fmt_note)

def col_letter(idx):
    s = ""
    i = idx
    while True:
        i, rem = divmod(i, 26)
        s = chr(65+rem) + s
        if i == 0:
            break
        i -= 1
    return s
