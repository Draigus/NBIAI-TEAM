| StartDate            | 2025-09-28                                     | ISO 8601 preferred |
| DueDate              | 2025-10-05                                     | ISO 8601 preferred |
| AssigneesEmails      | a.user@nbi-consulting.com;b.user@nbi-consulting.com | Semi-colon separated emails |
| Priority             | Low, Normal, Important, Urgent                 | Optional |
| LabelsCSV            | Design, Milestone                              | Comma separated label names used in the plan |
| ChecklistCSV         | Write outline|done=false; Get approvals|done=true | Items separated by semicolons. Each item as `Title|done=true/false` |

Save in OneDrive or SharePoint where Power Automate can read it.

## 1.3 Build the flow once, then reuse

In Power Automate:
1) Create flow, type: Instant cloud flow.
2) Trigger: **Manually trigger a flow**.
3) Action: **Get file content** for the selected Excel file, or use **For a selected file** as the trigger if you prefer right-click in SharePoint.
4) Action: **List rows present in a table**  
   - Location: your document library  
   - Document: the XLSX  
   - Table: `Tasks`

5) Action: **List buckets** (Planner) for the target Plan. Store in variable:
   - Add **Initialize variable**  
     - Name: `Buckets`  
     - Type: Array  
     - Value: output of List buckets `value`

6) Action: **Apply to each** → `value` from List rows. Inside the loop do:

   a) **Get or create Bucket**
   - **Compose** → `BucketName` = `items('Apply_to_each')?['Bucket']`
   - **Filter array** over `Buckets` where `item()?['name']` equals `BucketName`
   - **Condition**: If length of Filtered is 0, create it:
     - **Create bucket** with Plan Id and Name = `BucketName`
     - **Append to array variable** `Buckets` the new bucket object
     - **Set variable** `CurrentBucketId` to the Create bucket Id
     Else:
     - **Set variable** `CurrentBucketId` to `first(body('Filter_array'))?['id']`

   b) **Create the task**
   - **Create a task** (Planner)
     - Group Id: target Group
     - Plan Id: target Plan
     - Title: `Title`
     - Bucket Id: `CurrentBucketId`
     - Start Date Time: `StartDate`
     - Due Date Time: `DueDate`
     - Priority: use this expression to map text to numbers Power Automate expects:
       ```
       switch(toLower(trim(items('Apply_to_each')?['Priority'])),
         'urgent', 1,
         'important', 3,
         'low', 0,
         2)
       ```
       The default `2` is Normal.

   c) **Assign people**
   - Add a **Compose** called `RawAssignees` = `items('Apply_to_each')?['AssigneesEmails']`
   - **Initialize variable** `Assignees` as Array =  
     ```
     split(variables('RawAssignees'), ';')
     ```
   - **Apply to each** over `Assignees`:
     - **Trim** each value with a Compose: `trim(items('Apply_to_each_2'))`
     - **Condition** skip if empty
     - **Get user profile (V2)** with the email
     - **Assign a task** (Planner)  
       - Task Id: from Create a task  
       - Assign to: User Id from Get user profile

   d) **Update task details** for Description, Labels, Checklist
   - **Update task details**  
     - Task Id: from Create a task  
     - Description: `items('Apply_to_each')?['Description']`
     - Labels mapping. Planner exposes Category fields. Use a switch to turn on up to six common labels. Example with two labels:
       - Category1:  
         ```
         contains(concat(',', toLower(items('Apply_to_each')?['LabelsCSV']), ','), ',design,')
         ```
       - Category2:  
         ```
         contains(concat(',', toLower(items('Apply_to_each')?['LabelsCSV']), ','), ',milestone,')
         ```
       Adjust names per your plan. If you use more than six labels, repeat through Category6 or adapt for the 25 label fields if present in your tenant.
     - Checklist: build JSON from `ChecklistCSV`. Use a Compose named `ChecklistJSON` with:
       ```
       {
         "item1": {
           "title": "@{split(split(items('Apply_to_each')?['ChecklistCSV'],';')[0],'|')[0]}",
           "isChecked": "@{equals(toLower(split(split(items('Apply_to_each')?['ChecklistCSV'],';')[0],'|')[1]), 'done=true')}"
         }
       }
       ```
       For multiple items, either:
       - Keep a small number and duplicate `item2`, `item3` with indexes 1, 2, 3, or
       - Replace with child loop that constructs an object using an **Append to string variable** pattern, then **Parse JSON**. If you prefer, keep it simple and skip checklist automation and paste into Description.

   e) **Attach related files** if needed  
   - If your Excel has a column with SharePoint relative paths in NBI, you can add **Update task details** References, but most teams just link the channel file as a tab after go-live.

7) Save, run against one small plan. Verify. Then run against the rest.

### Notes and pitfalls
- Planner comments do not migrate as comments. If you must preserve them, paste into Description, or attach an HTML export to each task.
- Date formats: prefer `yyyy-MM-dd`. If your export is locale formatted, wrap with `formatDateTime(...)`.
- Buckets are per plan. The flow creates any that are missing.
- Priority mapping varies by connector version. The switch above covers common cases.
