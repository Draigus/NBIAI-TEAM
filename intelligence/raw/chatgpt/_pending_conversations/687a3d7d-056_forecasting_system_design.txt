I dont understand what this means "MAPE ≤ ±3 % on back‑cast"

[ASSISTANT]
**MAPE ≤ ±3 % on back-cast** is just a quality-bar statement for the Dynamic Metric Engine:

| Term | Plain-English meaning |
|------|----------------------|
| **MAPE** (Mean Absolute Percentage Error) | A common accuracy metric for forecasts. We look at the absolute % difference between what the model predicts and what actually happened, average that across all days (or weeks, months, etc.), and express it as a percentage. |
| **Back-cast** | Instead of predicting the future, we run the model **on a period where we already know the real results** (e.g., the last 6 months of Sarge Universe soft-launch data). This lets us judge how well the model would have performed. |

So **“MAPE ≤ ±3 % on back-cast”** means:

> *When we replay the model on historical data, the average absolute difference between the model’s numbers and the real numbers must be no more than 3 percent.*  

If the back-cast error is larger than 3 %, the team must tune the model (change curves, add variables, etc.) until it meets or beats that accuracy threshold before we call the Dynamic Metric Engine “done.”
