# Excel Budget Calculator Template for Live-Services Game Development

This template helps a **live-services game development team** estimate the time and cost to produce various game assets. It is structured for clarity and usability, with sections for different asset categories and development roles. The design emphasizes clear inputs for workdays, automatic cost calculations, and summary totals, all organized in a reader-friendly layout.

## Asset Categories and Subtypes

Organize the spreadsheet by **asset categories**, each with its own subsection for asset subtypes. This hierarchy makes it easy to focus on one content area at a time and roll up totals per category. For example, asset categories and subtypes might include:

- **Cars** – subtypes such as *Off-road*, *Rally*, *F1*, *Touring*, etc.  
- **Tracks** – different track types or environments (e.g., *City Circuit*, *Off-road Track*, *Stadium*).  
- **Engine Components** – e.g., *Engine*, *Transmission*, *Brakes*, *Suspension*, etc.  
- **Body Kits** – e.g., *Bumpers*, *Spoilers*, etc.  
- **Interiors** – (various interior designs or components for vehicles).  
- **Visual Mods** – e.g., *Lights*, *Charms*, *Bobble Heads* (cosmetic add-ons).  
- **Performance Mods** – e.g., *Turbo*, *Drivetrain*, *Clutch*, etc.  

Each **asset subtype** will be a line (or set of lines) under its category where team members input the effort required. Group the rows for each category together, and consider using Excel’s **Outline** grouping feature to make these sections collapsible【6†L88-L96】. For instance, you can have a bold header row for "Cars" and indent the Off-road, Rally, F1, Touring rows beneath it. This way, you can collapse the category to a single summary line when needed, which *improves readability* and allows quick per-category subtotals【12†L143-L151】.

At the end of each category section, include a **Category Summary** row (e.g., "Total Cars"). This summary will sum up the total workdays and total cost for all subtypes in that category. With Excel grouping, the summary can serve as the visible line when the details are collapsed, letting you see high-level totals per category at a glance【12†L143-L151】. 

## Development Roles by Department

Across the columns (horizontally), list all the development roles involved in asset production. It’s helpful to **group roles by department** with clear headers or shading to distinguish each group. You can, for example, have a header row that spans the columns for each department (using merged cells or a top-row label). The roles provided, grouped by department, include:

- **Art Department:** *Character Artist*, *Animators*, *Riggers*, *Concept Artists*, *Weapon/Equipment Modelers*, *VFX Artists*, *UI Artist*, *Environment Artist*, *World Object Modelers*, *Technical Artists*, *Art Director*.  
- **Design Department:** *UI Designers*, *Audio Designers*, *Level Designer*, *Game Mode Designer*, *Character Designer*.  
- **Engineering Department:** *Gameplay Engineers*, *UI Engineers*, *Build Engineer*, *Backend Engineers*, *Network Engineer*.  
- **Production & QA:** *Art Producer*, *Design Producer*, *Engineering Producer*, *QA* (Quality Assurance).  

Each role will have a column for their estimated work on each asset subtype. For clarity, group or separate these columns by department (you might use Excel’s column grouping to collapse entire departments if the sheet becomes too wide【12†L184-L192】). For instance, you could hide the Engineering columns when focusing on art/design, and so on. However, even without collapsing columns, clearly labeling the departments (e.g., a tinted header cell reading "Art Roles" across the art role columns) will help users navigate the many roles.

> **Extensibility:** The template can accommodate additional roles as needed. If your project adds a new role (e.g., a *Lighting Artist* or *Community Content Designer*), insert a new column under the appropriate department and include it in the calculations. The structured layout ensures you can expand the role list without breaking the overall format.

## Input Fields and Assumptions

The primary inputs in this sheet are the **workdays required** by each role for each asset subtype. These will be entered in the intersecting cells of the asset row and role column (e.g., under *Character Artist* for *Off-road Car*, you might input **5** to indicate 5 workdays of character art are needed to create an Off-road Car asset). Key inputs and assumptions include:

- **Workdays per Role per Asset:** Every role column under an asset subtype row is an editable cell where the respective team member estimates the number of **workdays** needed for that role to produce the asset. All these cells are user inputs. They should be clearly marked (for example, with a specific background color) as **editable** values. Each cell will typically contain an integer or decimal representing days of work (e.g., 1.0 for one day, 0.5 for a half-day).  
- **Average Annual Salary (per Role):** At the top of the sheet (or in a separate **Assumptions** section), include a cell for the **average salary per role per year**. The default can be set to **£85,000**, as specified, but this cell should be editable in case the average salary changes or you want to input a different figure. This is a global assumption applied to all roles (alternatively, you could allow different averages per department, but the simplest approach uses one average).  
- **Workdays per Year:** In the same assumptions section, have a cell for the number of **workdays per year** used for calculations. It's given as **220 workdays/year**, which accounts for weekends, holidays, and leave. (220 is a common estimate for workable days in a year after excluding non-working days【25†L360-L368】.) This too can be adjusted if needed (for example, some teams might use 250 or 200 depending on overtime expectations or additional time off).  

These assumption inputs (salary and workdays/year) should be clearly labeled and placed prominently (often at the top-left of the sheet for visibility). By placing key inputs at the top or left, and results toward the bottom or right, you follow a layout convention that improves usability【16†L163-L171】. For instance, you might create a small **Parameters** box in the top-left corner of the spreadsheet that says: 

- *Average Salary (per role, per year)* = **£85,000** (editable)  
- *Workdays per Year* = **220** (editable)  
- *(Optional)* Cost per Workday = **£385** (calculated automatically from the above, see below)

Label these clearly (e.g., "Assumptions:"), and possibly give them a distinct cell style. Any cell meant for user input (like these assumption fields or the workday estimates in the main table) should be formatted in a special way to **indicate editability** – for example, a light fill color or blue text. A common spreadsheet convention is to color-code inputs vs. calculations (e.g., blue font or shading for input cells, and black font or grey-shaded cells for formulas)【15†L192-L198】. This visual cue helps users quickly identify where they can enter data. You might also protect the sheet (locking formula cells) to prevent accidentally overwriting calculations【16†L167-L171】, while leaving input cells unlocked.

## Cost Calculation per Role

The template will **automatically calculate the cost** for each role’s contribution to an asset once the workdays are entered. The cost per role per asset is based on the input days multiplied by that role’s daily rate. Given the assumptions:

- **Daily Rate per Role:** Calculate this from the average salary and workdays/year. For example, with £85,000/year and 220 days/year, the **daily cost** per role is approximately £85,000 / 220 ≈ **£386** per day【25†L360-L368】. (This cell can be shown in the assumptions section as mentioned, or it can be hidden in the formula; showing it is useful for transparency.) 

Using the daily rate, each role’s cost for a given asset subtype is: 

```
Role Cost = (Annual Salary / Workdays per Year) * Role Workdays Input
``` 

For instance, if *Animators* need **10 days** to create animations for an Off-road Car, and the daily rate is ~£386, then the **Animator cost** for that asset is 10 * £386 = **£3,860** (approximately). All such calculations should update instantly when a user changes the workdays or adjusts the salary assumptions. It’s best to have a formula in each relevant cell to do this multiplication automatically once days are entered. 

**Implementing the formula:** You can set up the sheet such that for each role column, there are two sub-columns: one for "Days" (input) and one for "Cost" (calculated). For example, column for Character Artist might actually be split into *Character Artist Days* and *Character Artist Cost*. In the "Cost" column cells, put a formula like `=Salary_per_year / Workdays_per_year * [Days Input]`. This way, when the user types in the days, the cost appears next to it. If you prefer to keep a single column per role (for compactness), you could show just the days in the main table and have the cost calculation in a tooltip or separate detailed section, but **showing the cost explicitly is clearer** for a budget sheet.

Ensure that **cost cells are formatted** as currency (£) with appropriate rounding (no cents needed if working in whole pounds). The formula references the assumption cells for salary and workdays, so that changing those assumptions will recalc all costs. (For example, if the salary average is increased, all role costs will rise accordingly.)

## Total Time and Cost per Asset

For each asset subtype (each row in the main table), the template should summarize the **total effort and total cost** across all roles:

- **Total Workdays per Asset:** Add up all the days entered by every role for that asset. This gives the combined development effort in workdays for that asset. You can have a dedicated column at the far right of the table (after all role columns) labeled **"Total Days"**. This cell will sum the row across all the role-day input cells. For example, if an asset requires 5 days from artists, 3 from designers, 2 from engineers, etc., the Total Days might be 10 (just an example). This sum updates as more roles add their estimates.  
- **Total Cost per Asset:** Similarly, calculate the sum of all role costs for that asset. You can have another column labeled **"Total Cost (£)"** to the right of Total Days. This cell will sum across that row all the calculated cost cells (or it could equivalently be `= Total_Days * Daily_Rate` if all roles use the same daily rate; however, summing the cost cells is more flexible, especially if different roles might have different rates in the future). 

Place these total columns adjacent to each other for easy reference (e.g., Asset Name | ... [roles] ... | **Total Days** | **Total Cost**). By inspecting these, the team can immediately see how many workdays an asset will take in total and what it will cost given the current inputs.

To ensure **summary rows per asset category** are meaningful, the category summary (like "Total Cars") should also aggregate these totals from all subtypes in that category:

- **Category Total Days:** Sum of Total Days of all assets in that category.
- **Category Total Cost:** Sum of Total Cost of all assets in that category.

These category summary rows could use Excel’s `SUBTOTAL` function or simple SUM formulas over the range of that category’s rows. If you use the grouping feature, Excel can auto-generate these subtotals when collapsing sections【12†L143-L151】. For example, the "Total Cars" row's Total Days cell = SUM of all Off-road, Rally, F1, Touring Total Days; likewise for cost. This provides a quick budget overview for each type of asset content.

You might also include a **grand total** at the bottom of the sheet (summing all categories) if desired, to see the overall budget for all listed assets. This can be helpful for a high-level budget view.

## Layout and Formatting for Clarity

Design the spreadsheet layout to be intuitive and easy to navigate. Here are some layout and formatting recommendations to enhance usability:

- **Section Grouping:** As mentioned, group the data by asset category. Use **bold headings** for category names, and indent asset subtype rows under each. Apply a subtle fill color or border to distinguish category headers. Leverage Excel’s outline grouping to make these sections collapsible【6†L88-L96】, so users can collapse categories they’re not interested in viewing in detail. When collapsed, only the category summary row will show, which makes the sheet much more compact and scannable when reviewing overall budgets per category. A plus/minus icon will allow expanding each category to see the individual assets (Excel automatically adds these outline symbols when grouping is set up). This structure **hides complexity** until needed and **improves readability** for large lists of assets【12†L143-L151】.  
- **Department Labels:** In the header row, label each department’s group of role columns. For example, have a merged cell spanning all Art roles labeled "Art", another spanning Design roles labeled "Design", etc. This header might sit on a second row above the individual role titles. If merging cells is inconvenient, simply give an extra row where you center the word "Art" above those columns. This way, it's immediately clear which columns correspond to which department. You can also color-code the column headers by department (e.g. a light green background for Art roles, blue for Design, etc.) to further visually separate them. This grouping in the header acts as a guide for the eye when entering data across many columns.  
- **Editable vs. Calculated Fields:** Clearly **differentiate input cells from formula cells**. As noted earlier, color-coding is very useful. For example, fill all input cells (workdays and assumption fields) with a light pastel color, and maybe use bold or blue text for them【15†L192-L198】. All formula-driven cells (costs, totals) could have a grey shaded background or just regular black text on white (since users shouldn’t type there). You might also italicize or lock formula cells. Adding a small note or legend (e.g., "*Blue cells = enter values; Grey cells = calculated*") in a corner of the sheet can be helpful for first-time users. This prevents confusion about where to input data.  
- **Frozen Panes:** Consider freezing the top rows and the first column for easier navigation. By freezing the header row, the role names and department labels will stay visible as you scroll down through many assets. Freezing the first column (which contains the asset category/subtype names) will keep the asset labels in view as you scroll horizontally across dozens of role columns. This way, you'll always know which asset and which role a particular cell refers to, no matter where you scroll. This simple Excel feature makes large spreadsheets much more user-friendly.  
- **Table Structures:** You can format the main area as an **Excel Table** (Insert > Table) which can automatically carry formulas down new rows and provides filtering options. However, be cautious with grouping and merged headers if using the Table feature, as those can interfere. Even without formally using an Excel Table, ensure consistent formatting of rows (alternating row shading can improve readability in a wide matrix of numbers).  
- **Labels and Units:** Label all totals and cost columns clearly with units (e.g., "Total Cost (£)"). Include a top-row label for the role columns like "Days" or "Workdays" and "Cost (£)" if you separate day and cost columns per role. It should be obvious which fields are days of effort and which are monetary. Also, in the assumptions section label the currency (e.g., "Average Salary (£/year)") and any other units. Clear labeling helps prevent user errors.  
- **Data Validation (Optional):** For input cells like workdays, you might apply data validation to ensure only numbers within a sensible range are entered (e.g., no negative days, maybe set a reasonable max like 100). This isn't strictly necessary, but it can guard against typos. Similarly, the salary and workdays/year fields could have validation (e.g., workdays/year should be a positive number, perhaps between 200 and 260).  

By following these layout practices, the resulting template will be easy to update and interpret. The combination of **structured sections, color cues, and Excel’s grouping** makes the budget calculator both comprehensive and navigable. Users can input their estimates for each role in the colored cells and immediately see the cost implications, while producers or leads can collapse categories to see high-level summaries.

        df["Asset Subtype"] = asset_list  # Fill in the asset subtype names
        
        # Fill formulas for role cost and totals
        for row_idx in range(len(asset_list)):
            excel_row = row_idx + 2  # Account for header row in Excel
            # Populate cost formulas for each role
            for idx, role in enumerate(roles):
                day_col_idx = 2 + idx * 2  # 'Asset Subtype' is column 1, so first role days start at column 2
                cost_col_idx = day_col_idx + 1
                day_col_letter = get_column_letter(day_col_idx)
                cost_col_letter = get_column_letter(cost_col_idx)
                
                # Formula: = [DayCell] * (Assumptions!$B$1 / Assumptions!$B$2)
                df.at[row_idx, f"{role} Cost"] = f"={day_col_letter}{excel_row}*(Assumptions!$B$1/Assumptions!$B$2)"
            
            # Total Days formula: sum of all 'Days' columns for that row
            days_col_letters = [get_column_letter(2 + idx * 2) for idx in range(len(roles))]

            df.at[row_idx, "Total Days"] = total_days_formula
            
            # Total Cost formula: sum of all 'Cost' columns for that row

            df.at[row_idx, "Total Cost"] = total_cost_formula
        
        # Write DataFrame to the sheet
        df.to_excel(writer, sheet_name=sheet_name, index=False)
    
    # Save the workbook

### How this sheet is laid out

1. **Assumptions (Top-Right Corner)**  
   - **Average Salary (£ per year)** (default 85,000)  
   - **Workdays per Year** (default 220)  
   These two cells feed every “Cost” formula in the sheet. If you need to adjust rates or working-days, just change these two values.

2. **Category Blocks (Left/Center)**  
   Each major asset category (Cars, Tracks, Engine Components, Body Kits, Interiors, Visual Mods) appears in its own block. Within each block:  
   - The very first row is the **category name** (for example, “Cars”).  
   - Below it, each **role** is a separate row (Concept Artist, World Object Modeler, Technical Artist, etc.).  
   - The columns to the right of the role name are **asset subtypes** (e.g., Off-road, Rally, F1, Touring for Cars).  
   - Under each asset-column, you see two cells for each role:  
     1. **[Role] Days** – where you type the number of days that role needs for that asset.  
     2. **[Role] Cost** – this is a formula: `=[DaysCell] * (Assumptions!$B$2 / Assumptions!$B$3)`  
        – i.e. Days × (Salary ÷ Workdays/year).  
   - At the far right of that transposed block are two extra columns:  
     - **Total Days** (sum of all “Days” cells in that row)  
     - **Total Cost** (sum of all “Cost” cells in that row)  

   Example (for the “Cars” block):  
   ```
   Row 1:  Cars
   Row 2:  Concept Artist  |  Off-road Days  |  Off-road Cost  |  Rally Days  |  Rally Cost  |  F1 Days  |  F1 Cost  |  Touring Days  |  Touring Cost  |  Total Days  |  Total Cost
   Row 3:  World Object Modeler |   (enter number)    |  =(cell)*(Assumptions!$B$2/Assumptions!$B$3) |   (enter)   |  … |  (enter)  |  … |  (enter)  |  … | =SUM(off-road, rally, F1, touring days) |  =SUM(off-road cost, rally cost, F1 cost, touring cost)
   Row 4:  Technical Artist    |   (enter)   |  … |  …  (same pattern)  … |  … |  … |  … |  … |  … |  …
   ⋮
   Row (roles+1):  Animator  |   (enter) |  …  ⋯
   ⋮
   Row (last role):  UI Artist  |  (enter) |  … | …
   ```

---

## 1. Structure and Naming

1. **Rename “Worksheet” tab to “Data Entry”**  
   - Make it crystal clear that this is where team members input their “Days” per role per asset subtype.  
   - Freeze the top header row and the first column (role names), so as you scroll through many asset columns you always see which role and which field you’re editing.

2. **Turn each major table into a proper Excel Table** (Insert → Table)  
   - Tables automatically expand when you add new rows or columns (e.g. adding a new asset subtype).  
   - You can use structured references (e.g. `=SUM(Table1[Off-road Days])`) rather than hard-coded ranges, which reduces formula errors if the layout shifts.  
   - Tables enable easy filtering by role, category, etc., without retyping filter criteria.

3. **Consistent Naming of Ranges**  
   - Create named ranges for key lists:  
     - Roles: `Role_List`  
     - Asset Subtypes: `Asset_List`  
     - Daily Costs: `DailyCost` (column in Assumptions)  
   - Anywhere you currently use `MATCH(“RoleName”, Assumptions!$A:$A, 0)`, you can replace that with `MATCH($A2, Role_List, 0)`, which is shorter and clearer.

---

## 2. Data Entry Automation

1. **Data Validation Dropdowns for Role and Asset Subtype**  
   - On “Data Entry,” column A (Role) should be a dropdown populated from `Role_List` (Assumptions sheet). That way, there are no typos (e.g. “Animtor” vs. “Animator”).  
   - Likewise, each “Subtype Days” column header can be generated from a dynamic named list of subtypes, or you can create a small validation dropdown in each row for “Asset Category” + “Asset Subtype” if you switch to a vertical layout. This approach becomes especially useful if you track every entry on a single “Transactions” table rather than a giant matrix.

2. **Switch to a “transactional” layout** (optional, but powerful)  
   - Instead of one giant grid where every role × subtype is a column, consider a normalized layout:  
     ```
     | Date | Asset Category | Asset Subtype | Role                  | Days |
     |------|----------------|---------------|-----------------------|------|
     |      | Cars           | Off-road      | Concept Artist        |   5  |
     |      | Cars           | Off-road      | World Object Modeler  |   3  |
     |      | Cars           | Rally         | Animator              |   4  |
     |      | Engine Components | Brakes     | Technical Artist      |   2  |
     | …    | …              | …             | …                     |  …   |
     ```
   - Each row is one “role working on one asset” entry.  
   - Benefits:  
     - You can add new asset subtypes or roles simply by inserting a new row.  
     - Summaries (per role, per asset, per category) become trivial via **PivotTable**.  
     - Filtering (e.g. “Show me only all ‘Animators’ on ‘F1’) is instantaneous.  
   - Once data is in this shape under a named Table (e.g. `WorkEntries`), you build a PivotTable on Sheet1 that automatically shows “Sum of Days” by Role vs. Asset Subtype, and also “Sum of Cost” by adding a calculated field:  
     ```
     Cost = [Days] * ( LOOKUP([Role], Assumptions!Role_List, Assumptions!DailyCost) )
     ```

     ```
     =SUM(Table_Data[@[Off-road Days]:[Touring Days]])
     ```

     ```
     =SUM(Table_Data[@[Off-road Cost]:[Touring Cost]])
     ```

By implementing these changes—especially moving to an Excel Table → PivotTable workflow—you’ll minimize manual formula upkeep, reduce errors, and give the live-service team a real-time view of resource allocations and costs in a few clicks.

   - We need to multiply each role’s hours by that role’s cost/hour, then sum across all roles for this subtype. The simplest way—without VBA—is a **SUMPRODUCT** that (a) checks for matching Category/Subtype, (b) multiplies Hours by the matching role’s Cost/Hour from Assumptions. In **E2**, enter:  
     ```
     =SUMPRODUCT(
       ('Asset_Roles'!$A$2:$A$500 = $A2) *            // match category  
       ('Asset_Roles'!$B$2:$B$500 = $B2) *            // match subtype  
       ('Asset_Roles'!$D$2:$D$500) *                   // Hours  
       (   INDEX(
             Assumptions!$E$2:$E$44,                    // Cost/Hour column
             MATCH('Asset_Roles'!$C$2:$C$500, 
                   Assumptions!$A$2:$A$44, 0)
           )
       )
     )
     ```  
     - In plain English: For each row 2:500 in Asset_Roles, if column A (Category) = $A2 AND column B (Subtype) = $B2, then take the Hours value (column D) and multiply it by the Cost/Hour found by MATCHing the Role (column C) in Assumptions. Finally, SUMPRODUCT adds these up.  
     - You can adjust `$D$2:$D$500` and the other ranges to cover more rows if you expect > 500 lines (just keep them consistent).  
   - Copy that formula down to E177.
