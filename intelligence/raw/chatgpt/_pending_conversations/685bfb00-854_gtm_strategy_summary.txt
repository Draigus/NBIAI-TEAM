The short answer: each time I run a Python block for you, it spins up a **temporary, sandboxed notebook** that lives just long enough to execute that single block. As soon as the block finishes—or hits a time-out or syntax error—that sandbox is torn down to keep everything clean, secure and resource-efficient. So:

1. **Fresh slate every call**  
   Nothing (variables, imports, files in working memory) persists from one Python call to the next unless I explicitly write a file to disk—e.g. the Word docs I saved into `/mnt/data/…docx`.

3. **Syntax / parsing errors**
