# Video Game KPI Forecast Model (Streamlit & Prophet)

We build a **local Streamlit app** that lets non-technical users upload historical game KPIs (e.g. DAU, ARPU, retention) and forecast them using Facebook/Meta’s **Prophet** library【1†L67-L73】【15†L135-L143】. Prophet is a free, open-source time-series forecasting tool that decomposes data into trend and seasonality components【1†L67-L73】. The app provides **daily, monthly, and yearly forecasts**, interactive Plotly charts, and an Excel export. All libraries (Streamlit, Pandas, Prophet, Plotly, etc.) are free and open-source【15†L135-L143】【17†L41-L49】. 

## Key Steps and Components

- **Libraries & Setup:** We use `streamlit` for the UI, `pandas` for data handling, `prophet` for forecasting, `plotly` for interactive charts, and `openpyxl` (or `xlsxwriter`) for Excel export.  Prophet can be installed via `pip install prophet` (it was formerly named `fbprophet`)【15†L135-L143】. Plotly for Python is MIT-licensed and free【17†L41-L49】.  
- **User Inputs:** In the Streamlit sidebar, we add a file uploader (`st.file_uploader`) for the CSV, a multiselect/dropdown (`st.multiselect`) for choosing one or more KPI columns to forecast, and a dropdown (`st.selectbox`) for the forecast horizon (1–5 years). Streamlit’s `file_uploader` widget handles CSV input easily【6†L369-L374】.  
- **Data Preparation:** After upload, we parse the CSV into a pandas DataFrame and convert the “date” column to datetime. We then let the user pick numeric KPI columns. For each chosen metric, we prepare a DataFrame with columns `ds` (date) and `y` (value) as required by Prophet.  
- **Forecasting with Prophet:** For each metric, we instantiate and fit a Prophet model (using default yearly/weekly seasonality). We create a future timeline for the chosen number of years (e.g. `periods = years*365` days) and call `model.predict(future)` to get the forecast. This yields daily forecasts (`yhat`, with uncertainty bands `yhat_lower`/`yhat_upper`).  
- **Aggregating Forecasts:** We extract the **daily forecast** directly from Prophet’s output. To get **monthly** and **yearly** forecasts, we group or resample the daily predictions by month and by year. For example, using pandas we can resample by month (`.resample('M')`) or group by year. This produces total/average forecasts per month/year.  
- **Interactive Plots:** We use Plotly to plot the historical data and forecast. For each metric we create an interactive line chart showing actuals vs. predicted values. Streamlit can display Plotly figures with `st.plotly_chart(fig)`【8†L219-L227】, which yields zoomable, hoverable charts for easy analysis.  
- **Summary of Findings:** For each forecasted metric, we compute key insights such as *peak forecast values* and *overall trend*. For example, we find the maximum daily forecast and its date, as well as total forecast per year to see growth or decline. These are shown as textual bullets.  
- **Export to Excel:** We write the forecasts to an Excel file using `pandas.ExcelWriter`. We create separate sheets (tabs) for each metric and frequency (daily, monthly, yearly). Pandas supports multiple sheets by specifying a unique `sheet_name`【18†L123-L131】. The user can download or save this Excel file.  
- **Local, No Backend:** Everything runs in the browser (locally) via Streamlit – no external servers or APIs are needed. The user launches the app with `streamlit run app.py` and all computation happens on their machine.

Below is a detailed code example (annotated with comments) that realizes this app. The code uses Streamlit’s sidebar for inputs and main page for outputs. It is designed for readability and ease-of-use by non-coders.

```python
import streamlit as st
import pandas as pd
from prophet import Prophet
import plotly.graph_objs as go
from io import BytesIO

# Title of the app
st.title("🎮 Video Game KPI Forecasting App")

# Sidebar for inputs
st.sidebar.header("1. Upload Data")
uploaded_file = st.sidebar.file_uploader("Upload CSV file with columns 'date' and KPIs", type=["csv"])
if uploaded_file is None:
    st.sidebar.warning("Please upload a CSV file.")
    st.stop()

# Read CSV into DataFrame
df = pd.read_csv(uploaded_file)
# Ensure 'date' column is datetime
df['date'] = pd.to_datetime(df['date'])

# Let user select which KPI columns to forecast
numeric_cols = [col for col in df.columns if col != 'date']
if not numeric_cols:
    st.error("No KPI columns found in CSV (expecting numeric columns besides 'date').")
    st.stop()
metrics = st.sidebar.multiselect("2. Select KPI(s) to forecast", options=numeric_cols)
if not metrics:
    st.sidebar.info("Select at least one metric to forecast.")
    st.stop()

# Forecast horizon in years
years = st.sidebar.selectbox("3. Forecast horizon (years)", [1, 2, 3, 4, 5], index=0)

# Prepare Excel writer in memory
output_excel = BytesIO()
writer = pd.ExcelWriter(output_excel, engine="xlsxwriter")

# Loop through each selected metric and forecast
for metric in metrics:
    # Prepare data for Prophet: columns ds (date) and y (value)
    df_metric = df[['date', metric]].rename(columns={'date': 'ds', metric: 'y'})
    df_metric = df_metric.dropna()  # Drop missing values
    
    # Train Prophet model on historical data
    model = Prophet(yearly_seasonality=True, weekly_seasonality=True, daily_seasonality=False)
    model.fit(df_metric)
    
    # Create future dataframe for desired forecast period (in days)
    days = years * 365
    future = model.make_future_dataframe(periods=days, freq='D')
    forecast = model.predict(future)
    
    # Extract daily forecast (dates and predicted value)
    forecast_daily = forecast[['ds', 'yhat', 'yhat_lower', 'yhat_upper']].copy()
    forecast_daily.rename(columns={'ds': 'Date', 'yhat': 'Forecast'}, inplace=True)
    
    # Compute monthly forecasts by summing daily predictions per month
    forecast_daily.set_index('Date', inplace=True)
    forecast_monthly = forecast_daily.resample('M')['Forecast'].sum().reset_index()
    forecast_monthly.rename(columns={'Date': 'Month', 'Forecast': 'MonthlyForecast'}, inplace=True)
    forecast_monthly['Month'] = forecast_monthly['Month'].dt.to_period('M').astype(str)
    
    # Compute yearly forecasts by summing daily predictions per year
    forecast_yearly = forecast_daily.resample('Y')['Forecast'].sum().reset_index()
    forecast_yearly.rename(columns={'Date': 'Year', 'Forecast': 'YearlyForecast'}, inplace=True)
    forecast_yearly['Year'] = forecast_yearly['Year'].dt.year
    forecast_yearly = forecast_yearly[['Year', 'YearlyForecast']]
    
    # Plot actual vs forecast
    fig = go.Figure()
    fig.add_trace(go.Scatter(x=df_metric['ds'], y=df_metric['y'], 
                             mode='lines+markers', name='Actual'))
    fig.add_trace(go.Scatter(x=forecast['ds'], y=forecast['yhat'], 
                             mode='lines', name='Forecast'))
    fig.update_layout(title=f"{metric} - Actual vs Forecast", xaxis_title='Date', yaxis_title=metric)
    st.plotly_chart(fig, use_container_width=True)  # Display interactive chart
    
    # Summary of key findings
    # Peak daily forecast and date
    peak_idx = forecast['yhat'].idxmax()
    peak_value = forecast.loc[peak_idx, 'yhat']
    peak_date = forecast.loc[peak_idx, 'ds'].date()
    # Growth from end of historical to end of forecast (percentage)
    last_history = df_metric['y'].iloc[-1]
    final_forecast = forecast['yhat'].iloc[-1]
    growth_pct = ((final_forecast - last_history) / last_history * 100) if last_history != 0 else None
    
    st.markdown(f"**Metric:** {metric}")
    st.write(f"- **Last historical value:** {last_history:.2f} on {df_metric['ds'].iloc[-1].date()}")
    st.write(f"- **Predicted final value:** {final_forecast:.2f} on {forecast['ds'].iloc[-1].date()}")
    if growth_pct is not None:
        trend = "increase" if growth_pct >= 0 else "decrease"
        st.write(f"- **Overall trend:** {trend} of {abs(growth_pct):.1f}% over {years} year(s)")
    st.write(f"- **Peak forecast:** {peak_value:.2f} on {peak_date}")
    st.write(" ")
    
    # Write to Excel sheets for this metric
    # Daily sheet
    forecast_daily.reset_index(inplace=True)
    forecast_daily.to_excel(writer, sheet_name=f"{metric}_Daily", index=False)
    # Monthly sheet
    forecast_monthly.to_excel(writer, sheet_name=f"{metric}_Monthly", index=False)
    # Yearly sheet
    forecast_yearly.to_excel(writer, sheet_name=f"{metric}_Yearly", index=False)

# Save Excel file to buffer
writer.close()
output_excel.seek(0)

# Provide download link
st.sidebar.header("4. Download Forecasts")
st.sidebar.download_button(
    label="Export all forecasts to Excel",
    data=output_excel.read(),
    file_name="kpi_forecasts.xlsx",
    mime="application/vnd.openxmlformats-officedocument.spreadsheetml.sheet"
)
```

**Explanation of Code:** The script starts by importing necessary libraries (all open-source: Streamlit for UI, Pandas for data, Prophet for forecasting, Plotly for charts).
