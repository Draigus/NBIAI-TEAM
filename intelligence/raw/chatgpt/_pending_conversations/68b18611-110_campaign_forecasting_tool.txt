## C) Core simulator equations and logic
Notation: everything is per date d, channel c unless otherwise noted.

1) Adstock and diminishing returns  
- adstock_c[d] = spend_c[d] + decay_c * adstock_c[d-1]  
- cpi_c[d] = cpi_floor_c + sat_alpha_c * (spend_c[d] / sat_beta_c)^(1/2) clipped to [floor, ceiling]  
- impressions_c[d] = 1000 * spend_c[d] / CPM_c[d]  
- clicks_c[d] = impressions_c[d] * CTR_c[d]  
- installs_paid_c[d] = clicks_c[d] * store_cvr_c[d]  
- organic[d] = baseline_organic[d] + uplift_per_paid_install * sum_c(installs_paid_c[d])  
- installs_total[d] = organic[d] + Σc installs_paid_c[d]

2) Creative fatigue and learning  
- CTR_c[d] = base_ctr_c * fatigue_factor_c[d] * seasonality[d]  
- fatigue_factor uses half-life: fatigue_factor[d] = exp(- cumulative_impressions_since_refresh / half_life_impr)  
- During channel learning periods, apply improving store_cvr_c[d] or reducing CPI via learning curve.

3) Cohorts, retention, and DAU  
- For each acquisition cohort k on day d: retained_users_k[t] = installs_k * R_t where R_t is the retention curve with tail defined by a parametric form (e.g., shifted Weibull or power-law), with channel and country modifiers.  
- DAU[d] = Σ over all past cohorts k of retained_users_k[age_of_k_on_d].

4) Monetization  
- payers_k[t] = retained_users_k[t] * payer_conv_profile_t  
- ARPPU_k[t] from priors or price points; IAP_revenue_k[t] = payers_k[t] * ARPPU_k[t].  
- Ads_revenue[d] = DAU[d] * ad_impr_per_DAU[d] * eCPM[d] / 1000.  
- Subscription revenue via cohortized attach and churn curves.  
- Total_revenue[d] = Σk IAP_revenue_k[t] + Ads_revenue[d] + Subs_revenue[d] + DLC[d].

5) LTV and ROAS  
- LTV_k[T] = Σ_{t=0..T} revenue_k[t] / installs_k.  
- ROAS_c[T] = (Σ revenue from cohorts sourced to c within T) / (Σ spend_c over those cohort dates).  
- Payback_days_c = min T where ROAS_c[T] ≥ 1.

6) CRM incremental lift  
- For segment s on day d: eligible_users_s[d] per rules.  
- Sends_s[d] = min(eligible_users_s[d], freq_cap constraints).  
- Delivered_s[d] = Sends_s[d] * deliverability_s.  
- Opens_s[d], Clicks_s[d], Conversions_s[d] via rates.  
- Incremental_lift_s[d] = (metric_treated − metric_control) using rolling holdouts.  
- Revenue_per_send_s[d] = incremental_revenue_s[d] / Sends_s[d].

7) Virality  
- referrals[d] = k_factor[d] * DAU[d-lag] with acceptance and duplication controls.

8) Risk and ops penalties  
- If downtime, scale DAU and revenue by uptime_ratio; spillover recapture optional.
