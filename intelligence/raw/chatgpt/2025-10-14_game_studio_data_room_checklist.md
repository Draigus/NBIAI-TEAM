# Investor-Grade Data Room Checklist for Game Studios

## Key Content
16-section data room structure: (1) Company overview with one-pager and runway. (2) Cap table with option pool and prior financing. (3) Team bios tied to shipped titles plus hiring plan. (4) Market and strategy with genre thesis. (5) Portfolio and pipeline with gate criteria. (6) Game design docs, economy docs, art bibles. (7) Tech: engine choice, CI/CD, code ownership, open-source audit. (8) Platform: dev kit status, TRC/XR compliance, store strategy. (9) Traction split by model: Premium (wishlists, demo metrics) vs F2P (CPI, retention D1/D7/D30, ARPDAU, LTV, ROAS). (10) Monetisation and economy. (11) Financials with forecasts and UA payback. (12) Legal: IP chain, middleware licences, GDPR/COPPA. (13) Contracts and vendors. (14) Operations and policies. (15) References. (16) Data room hygiene with index and versioning.

Round definitions: Seed = team, thesis, prototype, prove fun. Series A = product-market fit, soft launch metrics trending. Series B = scale what works, predictable KPIs. Series C = institutional scaling, M&A readiness.

## Decisions / Insights
- Glen decided: data room must distinguish Premium vs F2P traction sections with different evidence standards
- Glen decided: round definitions must be games-specific, not generic startup definitions
- Glen observed: IP ownership chain, middleware licences, and contractor IP assignments are frequently missing from game studio data rooms
