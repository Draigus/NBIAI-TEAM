# CH OKR Calibration and Long-Range Planning Framework

## Key Content

Org Discussion between Glen and Aris, 16 June 2026.

**OKR threshold calibration (agreed approach):**
- 1 week late = green (within normal variance)
- 4 months late = red (has consumed an unplanned quarter of spend)
- Internal plan uses a buffer; the external message only flags when that buffer is being consumed
- Goal: prevent red/yellow/green becoming meaningless because thresholds were never defined

**Long-range plan (LRP) milestones needed:**
Three domains: revenue, production quality, investor confidence. The LRP must cover:
- Platform expansion timing decisions (Xbox, PlayStation, mobile)
- Second-game timing
- These decisions are interdependent: platform expansion affects revenue trajectory, which affects second-game capital availability

**Studio IT and security posture (under Aris):**
Security posture explicitly linked to insurance and funding rounds — a breach can void both simultaneously. Areas: hardware budgets, breach prevention, GDPR/PII management, DPO appointment. This is operational governance, not just IT.

**PM role under Aris (decided needed):**
Function: close loops, run the red/yellow/green OKR dashboard, manage headcount planning. Frees Aris from chasing follow-ups and program-manages across all studio workstreams. Product Analyst role (Stavros Kilo identified as candidate) is first hire under Glen's marketing org, ahead of Marketing, BD, Web, Community, Brand and PR.

## Decisions / Insights

- [Glen/Aris] agreed: OKR thresholds must be defined (1 week = green, 4 months = red); vague RAG status is operationally useless
- [Glen/Aris] agreed: internal buffer vs. external message creates a two-layer status model — team sees honest state, investors see flagged state only when buffer is consumed
- [Glen] decided: PM role under Aris needed immediately; it frees Aris from ops chase work
- [Glen/Aris] agreed: security posture is a prerequisite for funding rounds and insurance, not an afterthought
- [Glen] identified: hiring sequence under Glen's org: Product Analyst → Marketing → BD → Web → Community → Brand and PR
