# CH Combat Playtest 2.0 — Pressure System, Weapon Identity, Enemy AI

## Key Content

Combat playtest review with Robin, Nadir, Vardis, Mustafa. Core combat: main hand (left click) + offhand heavy attacks (right click). Pressure system: striking enemies builds "cracks" stacks (1–7); detonating at different levels creates varying stun effects (Level 2: flinch, Level 5: stagger, Level 7: knockdown). Heavy attacks break blocks and apply status effects. Co-op opportunity: one player builds pressure, another detonates for area damage. Weapon type is primary differentiator for combat, progression, and abilities. Quick weapon swap (X key) strategic layer. Defensive actions (block/dodge/parry) driven by offhand weapon type. Two-handed weapons: transformation weapons (Bloodborne model — e.g. hammer with rocket charge). Enemy AI: tag-based system using 5 D&D attributes with 3 tiers each → 15+ archetypes. Modifiers add bespoke identity ("stronger together" for skeletons). Constitution = flinch interruption; Strength = knockback; Dexterity = mid-strike cancel/player tracking. Combat tone: realistic not anime — God of War or martial arts reference. Weapon combinations affect locomotion (shield carrier: heavier gate; dual daggers: more agile swagger). Questions for productionalisation: (1) does back-and-forth structure fit the game; (2) is fundamental system solid for building upon.

## Decisions / Insights

- [Nadir] decided: pressure system (crack stacks 1–7) as the core combat loop driver
- [Glen/team] decided: combat tone is realistic martial arts, not anime — God of War reference
- [Nadir] designed: 5-attribute D&D tag system for enemy AI with 15+ archetypes
- [Glen] flagged: animation team collaboration needed before production — current animations are marketplace placeholders
- [Glen/Robin] decided: move forward with base combat layer once team validates rhythm; no more iteration
