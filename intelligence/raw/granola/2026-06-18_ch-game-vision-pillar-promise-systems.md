# CH Game Vision: Pillar -> Player Promise -> Systems Architecture

## Key Content
Glen and Valeria agreed on a three-tier vision documentation structure: (1) pillars define design intent; (2) player promises translate pillars into player-facing experience statements ("as a player, I want combat to feel like I'm growing in skill"); (3) systems and content deliver the promises. Full structure to be published on the Confluence home page, visible to the entire team. Goal: any team member can check a feature against the vision and push back if it doesn't fit.

Current build assessed as "generic" and indistinguishable from other MMOs despite strong underlying design intent. Diagnosis: communication and documentation failure, not a design failure.

**CH game differentiators identified:**
- Combat: tab/skill-based with ground-based directional vectors; described as rare in MMOs.
- Crafting: collaborative construction mechanic -- a friend's presence when building a relic boosts completion chance by approximately 20%.

Simon Woodroffe tasked with reviewing the vision direction and working through this layer before pillar publication.

**Pillar locking process:** Glen and Robin to pre-align with Vardis before Simon is brought in. Once locked, published to the studio as "biblical commandments." Simon's input wanted but Vardis and Glen "paint the colours" first. Five versions currently in circulation causing studio confusion.

## Decisions / Insights
- Glen decided: pillars must be locked via Glen/Robin/Vardis pre-alignment before Studio publication.
- Glen/Valeria concluded: the generic appearance of the current build is a documentation failure, not a design failure.
- Glen concluded: the vision structure must be publicly visible so any contributor can self-check their work against it.
