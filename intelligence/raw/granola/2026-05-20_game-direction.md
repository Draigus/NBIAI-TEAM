# Game Direction — Weapon Forging System Design and Development Mandate

## Key Content

Foundational development mandate established: "Thou Shalt Not Hack" — no throwaway code, no fragile code. All code must be real, modular, reusable. Placeholder systems visually acceptable but architecture must be production-quality. Tech team leads determine build order, not creative director — systems layer on each other. Vertical slice target end of August. Scope reduction priority: add headcount first, explore outsourcing second, cut features only last. Weapon forging system designed in detail: 4 components (pommel, hilt, tang, blade), blade determines weapon type with different swing animations. Permanent creator attribution — first player to assemble a specific combination is recorded forever and can name the weapon (profanity filter). Breakdown mechanics: RNG quality degradation, minimum 1 component returned, primary enchantment preserved. Crafting tiers from gray (anyone) to gold (master crafters with exclusive abilities). Core design pillar formally named: "Forge Masters must always be valuable to the guild and ecosystem." Four-tier live service events: small (weekly), medium (bi-weekly), large (monthly), XL (seasonal). Monetisation: cosmetics only, no pay-to-win.

## Decisions / Insights

- [Glen] decided: "Thou Shalt Not Hack" — all vertical slice code must be production-quality architecture
- [Glen] decided: tech leads determine build order, not creative director — systems dependency drives sequence
- [Glen and Vardis] decided: permanent creator attribution in weapon system — first assembler recorded forever
- [Glen] decided: scope reduction hierarchy — headcount first, outsourcing second, feature cuts only last
- [Glen] decided: cosmetics-only monetisation — no pay-to-win weapons
