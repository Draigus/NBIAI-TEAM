# Producer as Cross-Department Defect Translator

## Key Content
The core problem in studios with "black boxes" -- departments operating without cross-department transparency -- is that request quality is never checked before work starts and output quality is never checked against criteria before sign-off. The result: assets delivered wrong, rework accumulates invisibly.

The producer's role is defined here as: (1) check ask details before work starts (does the request have enough detail to execute?); (2) check output against delivery criteria before sign-off (does the output match what was asked?); (3) track defects in retrospectives; (4) feed rework back into the producer backlog.

The escalation model: one defect = human, flagged internally. Repeated defects from the same person = escalate to their director. Persistent pattern = escalate to fractional head of studio / COO.

Exemplar: a glider asset went wrong because a lead declared it approved without the art director or game director's sign-off, and rigging started prematurely. Root cause: no defined request criteria, no definition of done, no visibility into asset progress.

## Decisions / Insights
- Glen decided: producer role is the structural fix for cross-department black boxes; not a project manager, a translator and accountability owner.
- Glen decided: the escalation ladder (defect → director → COO) creates a clear accountability chain that doesn't require the COO to manage every incident.
- Pattern recognised: leads operating in "talk to me one-on-one" mode without documented handoffs consistently deliver wrong assets weeks later.
- Glen observed: the producer must own request quality before work starts, not just delivery status.

## Context
1:1 with a studio COO, 2026-06-19. Studio is ~70 people; two producers (art and tech) have just accepted offers. The COO had raised concerns about leads blocking cross-department visibility deliberately. The Glider incident is from the preceding week.
