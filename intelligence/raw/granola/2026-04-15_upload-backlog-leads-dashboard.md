# Upload Backlog & Leads — Dashboard Feature Requests

## Account Management
- Wiped all custom permissions from Magnus's account to normal, then added admin. Hard-coded permissions removed — account now follows normal path.

## Dashboard UI Issues
- Too much data on dashboard view — need to reallocate and avoid duplication with workload page.
- Dashboard purpose: summary by client (portfolio view) or by individual client (filtered view). Show high-level work by project, blockers/reds, who is working on it.
- Circle chart on distribution card "doesn't make any sense visually."
- Status charts should stack by largest field first or allow click-to-restack.
- UI scaling problems across different monitor sizes (3440x1440 vs 2560x1080). Proposed: UI scaler setting like video games.

## Feature Requests
- Client profile section needs to be editable. Leads page filtering broken.
- Status fields on workload need "waiting on client" parameter.
- **Hiring page enhancements:** Each stage needs owner assignment (multiple owners). Interview phase: multiple owners can enter notes, "Hire"/"No Hire" button per interviewer.
- Replace "hired" stage with "onboarding." Onboarding card: inherited fields (name, role, start date, LinkedIn, client), auto-create P1 task for practice lead (Glen for games, Tom for org health), follow-up task to Magnus, file upload for onboarding docs.
- Alert button: move from floating to persistent bar, 30ms flash, maintain issue count.

## SOW Upload Feature
- System automatically strips pricing, fees, legal terms, confidentiality, IP clauses. Only scope stored, original PDF never saved. Admin only.

## Intelligence Value
Detailed feature backlog for NBI Dashboard from real user testing with Magnus. The hiring page enhancements (multi-owner interviews, auto-onboarding tasks) are significant workflow improvements. SOW upload with automatic clause stripping is a novel feature.
