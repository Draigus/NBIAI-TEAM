# Game Design Hierarchy: Pillars → Player Promises → Value Creation

## Key Content
A three-layer design hierarchy being built to give the whole studio a shared frame of reference for their work:

1. **Pillars**: the core design principles of the game. Multiple conflicting versions existed in this studio; the fix is to compile all versions, reword into something durable, get the CEO to approve, then lock.

2. **Player promises**: what the player feels as a result of each pillar being delivered. Moves from abstract principles to concrete experience statements.

3. **Value creation**: how each promise translates to measurable outcomes (retention, revenue, engagement). Every animator, designer, or engineer should be able to trace their work back through this chain: my task → pillar → player promise → value.

The framework is built on Miro. Goal: anyone in the studio can articulate why their current task matters in terms of player experience and business outcome.

Worked example to be built by leadership: one pillar traced all the way through player promise to value creation, to model the methodology for the team.

## Decisions / Insights
- Glen decided: compile all conflicting pillar versions into one master, get CEO sign-off, then lock -- no further versioning.
- Glen decided: build a worked example in Miro showing one pillar → player promise → value creation before sharing with the team.
- Pattern recognised: without a traceable hierarchy, individual contributors cannot connect their work to the product's strategic intent.
