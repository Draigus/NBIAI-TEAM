## Outstanding Feature Request (Next Session)

Glen wants an **artifacts/deliverables system**:

> "When the AI team is working on an output and finishes it, I want it to go into a section or page for each client for review. Once it's marked review, I want to be able to save any artifacts to the project under that task or project name. I want all the artifacts to be stored."

This means:
1. A new "Deliverables" or "Artifacts" tab/section per client
2. When the NBI AI Team (Paperclip) completes a task, the output gets surfaced in the dashboard under the client
3. Artifacts have a review workflow: Pending Review > Reviewed > Saved
4. Saved artifacts are stored per project/task with the ability to view/download
5. This likely needs polling the Paperclip API for completed issues and pulling their documents/work products

### Implementation approach:
- Add `artifacts` array to localStorage data model (per task)
- Add a "Deliverables" tab in the main navigation
- Poll Paperclip `/api/companies/{id}/issues?status=done` for completed work
- Pull documents from completed issues via `/api/issues/{id}/documents`
- Display in a review queue per client
- Allow marking as reviewed and saving to the task
