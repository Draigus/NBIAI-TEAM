Key Design Points

  • Distraction-free — no sidebar, no kanban, just the interview
  • Category progress bars show completion at a glance
  • Score buttons with labels (1=Poor through 5=Excellent)
  • Question dot navigation shows scored (teal) vs pending (grey)
  • Auto-saves on every interaction — no lost work
  • Responsive for tablet use in interview rooms

Card Elements

Everything visible on the proposed card

Always shown
  • Avatar — coloured initials circle (hue from name hash)
  • Name — 14px bold, truncated if long
  • Role — 12px muted
  • Stage badge — colour-coded pill
  • Days in stage — "5d", "2w" — amber after 7d, red after 14d
Shown when available
  • Source badge — LinkedIn / Referral / Agency / Job Board / Direct
  • Assignee faces — overlapping initials circles (max 3 + count)
  • CV indicator — document icon if has_cv
  • Comment count — speech bubble + number
  • Action border — left border: red=needs action, amber=stale, green=on track

New Database Fields Required

candidates.sourceTEXT — enum: linkedin, referral, agency, job_board, direct, other
candidates.stage_changed_atTIMESTAMPTZ — set on stage change, used for days-in-stage calc
candidate_comments tableid, candidate_id, author, body, created_at — same pattern as bug_report_comments
interviews tableid, candidate_id, interviewer_name, type, scheduled_at, duration, location, notes, outcome

Each Block Shows

On the block
  • Candidate avatar — initials circle

  • Interview type — Phone Screen / Technical / Cultural / Final
  • Time + location — "9:00 – 10:00 · Zoom"
  • Interviewer avatar — small circle, bottom-right
On click (opens panel)
  • Full candidate detail panel (existing)
  • Interview notes field
  • Outcome selector (Pending / Passed / Failed / No-show)
  • Reschedule / Cancel actions
  • Conflict warning if interviewer double-booked
