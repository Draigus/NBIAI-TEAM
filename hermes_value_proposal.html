The Fundamental Problem Hermes Solves

Context Resets Kill Momentum

Every session starts with "what's the state of X" rather than building on known ground. Multi-session projects (5-10 sessions) lose threading between decisions.

  • Couch Heroes org structure evolves weekly — lost between sessions
  • Goals competitive data captured once, stales immediately
  • Investor pipeline state (who responded, what's next) not tracked
  • Decision cascades (A constrains B, C, D) too complex to hold without persistence

No Proactive Intelligence

You come TO the tool with questions. Nothing surfaces relevant information before you ask. Opportunities and risks are invisible until you manually check.

  • BD leads go cold — nobody alerts you
  • Competitor pricing changes go unnoticed for weeks
  • Approaching deadlines not anticipated
  • Industry moves (layoffs, funding, launches) only visible when you search

Benchmark Data Stales Instantly

Your competitive intelligence, pricing data, market sizing, and AERM benchmarks are point-in-time snapshots. The moment you capture them, they're ageing.

  • 315-row Goals MTX dataset: snapshot from 21 April, already stale
  • AERM benchmarks require freshness verification every invocation
  • Revenue multiples shift quarterly with public earnings
  • No automated pipeline to refresh — it's all manual session work

Compounding Knowledge Doesn't Compound

20 years of gaming expertise lives in your head. The AI gets fragments of it per session. There's no mechanism for your institutional knowledge to accumulate in a usable form.

  • Salary benchmarks (5MB SalarySage) reloaded from scratch each time
  • Previous engagement patterns (what worked for which client type) not retrievable
  • Pipeline design templates not accumulating from real usage
  • Gaming practice context (557 lines) = big context load every session

Where Hermes Does NOT Add Value

  • Any work that requires Glen present in real-time — Interactive strategy sessions, decision-making, UAT, client calls. Hermes can prepare for these but cannot replace the human loop.
  • Deep coding sessions — Feature builds, bug batches, architecture work. Claude Code's skill ecosystem (TDD, writing-plans, parallel agents, 7-step pipeline) is calibrated. Hermes isn't a copilot.
  • Creative deliverable production — Pitch decks, pricing models, org charts, SOWs. These need iterative refinement with Glen's feedback. Claude Code handles them well.
  • One-off research you need right now — When you ask a question and want an answer in 30 seconds, Claude Code is already there. Hermes adds value to recurring/scheduled research, not ad-hoc queries.
  • Visual IDE integration — No inline autocomplete, no VS Code panel, no diff viewer. It's a background agent and chat interface, not a coding assistant.
  • Anything that needs to run while both PCs are off — Hermes on your hardware requires at least one machine running. If both are down, jobs don't fire. (For you: irrelevant since both are always on.)

Honest Trade-offs

One-Time Setup Effort

WSL2 configuration, systemd services, Task Scheduler persistence, Discord bot setup, Hermes install and profile configuration. One afternoon with a detailed implementation plan.

Maintenance Burden

Low but non-zero. WSL clock drift after sleep, Windows Defender exclusions, periodic Hermes updates. Less than an hour per month once running.

Security Configuration

Default Hermes config is allow-all. Needs hardening: Discord bot token management, channel permissions, credential storage for any comms monitoring (Slack/Teams tokens).

Max Plan Daily Limits

Each Hermes→Claude Code CLI call counts against your Max daily usage. For 5-15 scheduled jobs/day plus Discord chat, you'd need to monitor whether you hit caps. If so, reduce cron frequency.

Both PCs Always On

The #1 reason people pay for VPS is uptime. Your machines are already always-on. Zero hosting cost.

Existing Max Subscription

The expensive part (Claude 4.6 inference) is covered. No new LLM spending required.

Complementary, Not Replacement

Your entire Claude Code workflow stays untouched. Every skill, pipeline, and process you've built continues working. Hermes fills gaps around it.

MIT Licensed, No Lock-in

Fully open source. No vendor dependency, no surprise pricing, no feature gates. If it stops being useful, uninstall it.
